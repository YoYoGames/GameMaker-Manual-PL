Język GameMaker

Do programowania gier można używać strony GameMaker Language na dwa sposoby: umieszczając bloki w celu wizualnego "napisania" kodu lub pisząc kod ręcznie.

Pierwsza opcja, GML Visual, jest najlepsza dla początkujących i hobbystów, którzy chcą szybko zacząć tworzyć własne gry.

Druga opcja, Kod GML, pozwala w pełni wykorzystać możliwości witryny GameMaker Language poprzez ręczne pisanie kodu.

Chociaż GML Visual jest łatwiejszy do rozpoczęcia pracy, GML Code jest nadal łatwym w użyciu językiem programowania z potężnymi funkcjami(funkcje, metody, struktury i konstruktory itd.).

Tworzenie skryptów wizualnych/kodowych

Podczas tworzenia nowego zdarzenia Object, nowej strony Script asset lub czegokolwiek innego, co powoduje utworzenie skryptu, użytkownik jest proszony o wybór między kodem GML a wizualnym G ML:

Oprócz możliwości wyboru między GML Code i GML Visual, okno to zawiera dwa pola wyboru:

  • Nie pytaj ponownie dla tego projektu: Gdy ta opcja jest włączona, ustawia opcję gry "Domyślny język skryptowy" na nowy wybór użytkownika, dzięki czemu nie jest on proszony o ponowny wybór dla bieżącego projektu.
  • Nigdy więcej niepytaj: Gdy ta opcja jest włączona, ustawia preferencję "Domyślny język skryptowy" na nowy wybór, dzięki czemu nigdy więcej nie zostaniesz poproszony o dokonanie wyboru dla żadnego projektu.
