Dowolna struktura danych

Ta akcja powoduje zwolnienie struktury danych z pamięci i powinna być wywoływana zawsze wtedy, gdy struktura danych nie jest już potrzebna, na przykład w zdarzeniu Destroy nawet instancji lub w zdarzeniu Room End, itp. Uwolnienie struktury danych oznacza, że żadne inne akcje nie można się do niej odwoływać, ponieważ nie będzie ona już istnieć, choć można to sprawdzić za pomocą akcji If Data Structure Exists.

Należy pamiętać, że jeśli struktura danych nie ma indeksu przypisanego do zmiennej o zasięgu globalnym, nie zostanie przeniesiona z jednej strony room na drugą. Na przykład, jeśli utworzysz instancję gracza, a struktura danych listy zostanie przypisana do zmiennej w tej instancji, lista będzie musiała zostać uwolniona, jeśli gracz zostanie zniszczony lub jeśli room zmieni się z instancją gracza (chyba że instancja gracza jest oznaczona jako persistent), więc instancja gracza będzie miała dwie akcje Free Data Structure dla obu tych zdarzeń. dla obu tych zdarzeń. Ważne jest, aby wszystkie utworzone struktury danych były niszczone w ten sposób, gdy nie są już potrzebne, aby zapobiec wyciekom pamięci, które mogą spowodować problemy z wydajnością lub nawet awarię gry.

Składnia działania:

Argumenty:

Argument Opis
Type Typ uwalnianej struktury danych (List, Map, Grid lub Stack)
Variable Zmienna zawierająca indeks struktury danych do zwolnienia

Przykład:

Powyższy kod bloku akcji pobiera rozmiar listy, do której odwołuje się zmienna globalna a następnie używa for loop, aby loop przez nią przejść i zniszczyć instancje o identyfikatorach przechowywanych na liście. Po zakończeniu loop lista jest zwalniana z pamięci.

Powrót: Działania związane ze strukturami danych
Następny: Dodaj do listy
