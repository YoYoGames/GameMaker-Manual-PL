Redaktor Script

W tym rozdziale omówiono edytor skryptów (zwany także edytorem tekstowym), w którym wpisuje się cały kod wymagany do działania gry. Edytor ten jest używany do tworzenia własnych stron script assets, a także do kodowania zdarzeń obiektów, dodawania kodu tworzenia pomieszczeń i wielu innych rzeczy.

W tej części opisano po prostu sposób działania edytora script, a nie sposób tworzenia strony script assets ani użycia kodu GML lub wizualnej wersji GML do stworzenia strony scripts. Więcej informacji na ten temat znajduje się w następnych rozdziałach:

  • Kod GML Scripts
  • Skrypty wizualne GML

Podczas tworzenia strony script asset możesz zostać poproszony o wybranie języka GML Visual lub GML Code. Więcej informacji można znaleźć w części Język GameMaker.

W edytorze script piszesz swój kod GameMaker Language, który jest wbudowanym językiem programowania używanym w programie GameMaker (więcej informacji na ten temat znajdziesz w rozdziale Język GameMakera ). Gdy już lepiej poznasz GameMakera i zechcesz w pełni z niego korzystać, warto rozpocząć naukę tego języka, ponieważ znacznie rozszerza on Twoje możliwości podczas tworzenia gier.

Nazwę script asset można zmienić, klikając prawym przyciskiem myszy na w Przeglądarce zasobów i wybierając Zmień nazwę, ale należy pamiętać, że nazwa script musi być zgodna z zasadami skryptów dla assets, a więc może być tylko alfanumeryczna, musi zaczynać się od litery, a jedynym symbolem, jaki może zawierać, jest symbol "_" pod paskiem. Nazwy edytora script w innych miejscach nie można zmienić i będzie on miał nazwę właściwą dla edytowanego wydarzenia, np. wydarzenie object.

Po otwarciu okna edytora script będzie ono miało następujące opcje i układ:

Zakładka Skrypt / Zdarzenia

Edytor script otwiera się w oknie z zakładkami na górze, co pozwala na umieszczenie wielu stron scripts lub wydarzeń w jednym oknie (można jednak zmienić to zachowanie w Preferencjach, aby dla każdej strony script lub wydarzenia utworzyć nowe okno). Możesz kliknąć na zakładce, a następnie przeciągnąć ją, aby zmienić kolejność zakładek, lub - jeśli wolisz - możesz wyciągnąć z bieżącego okna i umieścić ją na stronie workspace, aby utworzyć nowe okno dla tej strony script (lub dodać ją do innego okna). Można także zmaksymalizować edytor script, aby utworzyć nową stronę workspace. Jeśli przeciągniesz kartę script poza okno IDE, zostanie utworzone nowe okno IDE, w którym będzie znajdował się zasób script i będzie można go używać tak samo, jak okna głównego.

Zwróć uwagę, że jeśli edytujesz kod ze zdarzenia obiektu w edytorze script i zmaksymalizowałeś edytor script lub masz go w osobnym oknie lub na stronie workspace, to menu edytora po kliknięciu prawym przyciskiem myszy będzie miało dodatkowe opcje:

  • Przywróć do przestrzeni roboczej: Spowoduje to wyprowadzenie edytora script ze stanu maksymalizacji/przestrzeni roboczej i ponowne powiązanie go z object w workspace.
  • Przejdź do Object: Spowoduje to przejście do strony workspace, na której znajduje się strona object z kodem, i skoncentrowanie się na stronie object.
  • Dodaj /Otwórz zdarzenie: Umożliwia dodanie nowego zdarzenia do strony object, do której należy bieżąca strona script, oraz otwarcie nowej karty kodu w edytorze Script dla dodanego zdarzenia. Jeśli w wybranym wydarzeniu znajduje się już kod, zostanie on zostanie otwarty w nowej karcie.

Informacje na temat pozostałych opcji menu , dostępnych po kliknięciu prawym przyciskiem myszy, znajdują się w części Edycja poniżej.

Rynna

Rynna jest używana do wyświetlania numerów linii kodu, a także do przekazywania pewnych informacji o stanie kodu. Co najważniejsze, jeśli zrobisz literówkę lub niepoprawnie skonstruujesz kod, GameMaker IDE poinformuje Cię o tym, oznaczając linię kodu czerwonym wykrzyknikiem , wskazującym na błąd składni. Wyszukaj/Zamień

Podczas pracy w edytorze script można nacisnąć przycisk / + "F", aby otworzyć okno wyszukiwania, lub / + "H", aby otworzyć okno wyszukiwania i zamiany:

W tym miejscu można przeprowadzić wyszukiwanie lokalne dla wprowadzonego słowa kluczowego, a po wprowadzeniu szukanego terminu można użyć strzałek w prawym górnym rogu okna wyszukiwania, aby przechodzić od jednego znalezionego terminu do następnego na stronie script. Sposób przeprowadzania wyszukiwania można zmienić, przełączając następujące przyciski:

Po włączeniu tej opcji funkcja wyszukiwania będzie podświetlać tylko te strings, które pasują do całego wprowadzonego tekstu string. Na przykład przy wyłączonej funkcji wyszukiwanie słowa "random" spowoduje wyświetlenie wszystkich słów, które zawierają to słowo string - np. irandom(), lub randomise() - podczas gdy włączenie tej opcji spowoduje wyświetlenie tylko funkcji random().
Gdy ta opcja jest włączona, mówisz GameMakerowi, aby sprawdzał nie tylko zawartość wyszukiwanej strony string, ale także jej obudowę. Na przykład, jeśli masz stronę sprite o nazwie "spr_Dog" i wyszukasz na niej " dog" z wyłączonym przełącznikiem, to sprite string zostanie podświetlony, ale jeśli przełącznik jest włączony , to nie zostanie, ponieważ "Dog" nie jest już uważany za to samo co "dog".

Jeśli okno wyszukiwania zostało otwarte za pomocą / + "F", funkcja zamień nie będzie widoczna, więc aby ją otworzyć, należy kliknąć przycisk Zamień . go. Po otwarciu okna można wprowadzić tekst string, który zostanie użyty do zastąpienia dowolnego wyszukanego tekstu string, używając następujących przycisków do wykonania akcji:

Kliknięcie tego przycisku spowoduje zastąpienie następnego wyszukanego terminu string znalezionego na stronie script podanym zamiennikiem string. Należy pamiętać, że za "następny" termin uważa się następny po aktualnej pozycji kursora, a do różnych terminów można przechodzić za pomocą strzałek znajdujących się w prawym górnym rogu okna wyszukiwania.
Kliknięcie tego przycisku spowoduje zastąpienie wszystkich przykładów wyszukiwania string w obrębie strony script przy użyciu podanego polecenia replace string.

Uwaga: jeśli chcesz przeprowadzić wyszukiwanie globalne (tzn. przeszukać cały projekt, a nie bieżący adres script lub wydarzenie), możesz nacisnąć / + + "F".

Główny edytor kodu

Główny edytor kodu to miejsce, w którym napiszesz cały swój kod do utworzenia strony script asset lub wypełnienia zdarzenia object. Samą edycję kodu omówiono w rozdziale Edycja, znajdującym się w dalszej części tej strony.

Informacje

Pasek informacyjny na dole strony IDE pokazuje numer wiersza, w którym aktualnie się znajdujesz, oraz pozycję w wierszu. Jest to również miejsce, w którym można zobaczyć pomocnik kodu, czyli wiersz tekstu, który pokazuje aktualnie edytowaną funkcję wraz z wraz z wymaganymi przez nią argumentami. W miarę wypełniania funkcji w edytorze argumenty będą podświetlane, aby pokazać, które z nich są aktualnie edytowane.

Należy pamiętać, że wszelkie opcjonalne argumenty w funkcjach script będą wyświetlane z nawiasami [] wokół nich. Jeśli użyto komentarzy JSDoc Script w ramach strony script asset, to informacje informacje, które podałeś, będą również wyświetlane w tym miejscu.
