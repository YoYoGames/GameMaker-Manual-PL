Redaktor Extension

Pakiet rozszerzeń może być użyty do dodania szeregu dodatkowych funkcji i stałych do GameMaker Language (GML) i łatwego przenoszenia ich między projektami. Funkcje Extension mogą być również powiązane z zewnętrznymi SDK lub kodem źródłowym napisanym w innych językach (jak Java dla Android, Objective-C dla iOS, itp.), które mogą być użyte do rozszerzenia funkcjonalności Twojej gry (zwykle używane do implementacji interfejsów API stron trzecich, takich jak dla reklam, zakupów w aplikacji i innych).

Rozszerzenia mogą być używane do tworzenia biblioteki funkcji, których często używasz we wszystkich swoich projektach, więc zamiast importować tę samą scripts do każdego projektu, możesz po prostu dodać jedną extension asset, która zawiera zbiór twoich ulubionych funkcji i stałych. Takie funkcje mogą być również specyficzne dla określonej platformy, a Ty możesz łatwo wybrać, na jakich platformach dana extension może być używana.

Przykład tworzenia extension krok po kroku można znaleźć na stronie: Creating An Extension

Aby wyłączyć extensions w zależności od platformy docelowej, zobacz: How to Disable Extensions?

Importowanie rozszerzeń

Zwykle strony trzecie extensions będą pochodziły z GameMaker Marketplace, a te, które zasubskrybowałeś, będą wymienione w oknie Marketplace Library, które możesz stamtąd zainstalować. Jeśli nie dostałeś swojego pakietu extension z Marketplace, nadal możesz dodać go do projektu klikając prawym przyciskiem myszy na gdziekolwiek w Asset Browser i wybierając Add Existing, a następnie przeglądając lokalizację rozszerzenia, by je wybrać.

UWAGA Jeśli masz pakiet .yymps/.yymp/.gmez, możesz przeciągnąć go do IDE, aby go zaimportować, lub przejdź pod menu "Narzędzia" i wybierz "Importuj pakiet lokalny".

Platformy docelowe

Po dodaniu extension do Asset Browser, powinieneś nadać mu nazwę i numer wersji, a także ustawić, na jakie platformy chcesz go eksportować. Poprzez odznaczenie wybranych platform, możesz powiedzieć GameMakerowi, by nie eksportował extension jako części pliku wykonywalnego dla tych platform. Powinieneś również upewnić się, że właściwości extension pasują do wybranych platform docelowych, a także wybrać kompatybilne cele eksportu dla plików extension w oparciu o następujące elementy:

  • Wszystkie cele: GML extensions. Są to extensions wykonane przy użyciu GameMaker Language (GML) i są zgodne ze wszystkimi platformami docelowymi. (Zauważ, że jest to jedyny typ extension, który obecnie obsługuje cel Opera GX. )
  • Windows (Standard, UWP, Xbox): Możesz użyć plików Windows DLL (Dynamic Link Libraries). Należy pamiętać, że jeśli zaznaczyłeś opcję "Use x64 Windows Runtime" w opcjach gry w systemie Windows, to wszystkie DLL dla projektu muszą być kompatybilne z 64-bitami. 32-bitowe biblioteki DLL nie będą działać z 64-bitowym programem wykonywalnym.
  • macOS: Cel macOS może używać standardowych plików DyLib dla extensions.
  • Ubuntu (Linux): Dla celu Ubuntu musisz upewnić się, że twój extension używa plików .so.
  • HTML5: Platformy te mogą korzystać z JavaScript extensions.
  • Android: Ta platforma wymaga napisania extension w Java lub Android Framework SDK.
  • iOS / tvOS: Extensions dla tej platformy są napisane w Objective C i wymagają pliku *.mm i *.h lub, jeśli używają jakichkolwiek frameworków, prekompilowanego natywnego pliku *.a.
  • PlayStation: W przypadku PlayStation pliki extensions powinny być plikami PRX, przy czym każdy cel wymaga innego PRX, aby dostosować się do specyfikacji celu (PS3, PS4 lub PS5). W przypadku pracy na PSVita, extension powinno być SUPRX.

UWAGA W przypadku niektórych witryn extensions wymagane jest przestrzeganie określonych konwencji nazewnictwa, aby GameMaker prawidłowo zidentyfikować platformę i architekturę procesora, dla której są przeznaczone. Więcej informacji można znaleźć w części Konwencje nazewnictwa.

Ponieważ extension może mieć różne powiązane pliki, możliwe jest stworzenie jednego extension dla wielu platform, w takim przypadku należy określić wszystkie kompatybilne platformy w głównym edytorze extension (tj.: zaznaczyć pole wyboru obok nich), a następnie w indywidualnych właściwościach plików dla każdej części extension, ustawić platformę, na którą te pliki mają być eksportowane.

Aby zmienić platformę docelową dla konkretnego pliku extension lub placeholder należy najpierw otworzyć właściwości extension (kliknąć dwukrotnie the extension w Asset Browser), a następnie po wejściu do właściwości extension wybrać z prawej strony plik zasobu, który chcemy edytować. Odkompresować przy budowaniu

Opcja "Uncompress on build" może być użyta dla skompresowanych plików archiwalnych, które są przeznaczone dla określonej platformy docelowej. Jeśli ta opcja jest włączona, zawartość takiego archiwum zostanie rozpakowana do katalogu Included Files zbudowanego pakietu, ale dopiero po skopiowaniu na platformę docelową.

Na przykład, jeśli włączysz to dla pliku .zip w swoim extension, a następnie rozpoczniesz kompilację macOS z Windows, to archiwum .zip zostanie skopiowane jako-is z Windows do macOS, a następnie zostanie nieskompresowane podczas kompilacji na platformie docelowej (którą w tym kontekście jest macOS).

Wywoływanie funkcji Extension

Po skonfigurowaniu swojej extension, możesz używać funkcji i stałych dla niej tak samo, jak wbudowanych funkcji i stałych używanych w GameMaker Language. Jeśli Twoja gra jest międzyplatformowa (na przykład jest przeznaczona do uruchamiania w systemie Windows i Android celach), to powinieneś użyć stałych i funkcji OS, aby sprawdzić bieżącą platformę przed wywołaniem extension specyficznych funkcji, aby zapobiec problemom, na przykład:

switch(os_type)
{
    case os_android:
        call_android_extension_function(x, y, z);
    break;

    case os_windows:
        call_windows_extension_function(x, y, z);
    break;
}

Zauważ, że dla różnych platform twój extensions może dzielić nazwy funkcji, więc jeśli masz extension z różnymi plikami do (na przykład) wyskakiwania wiadomości na Windows i dla Android, oba pliki extension mogą dzielić te same nazwy funkcji, o ile oba są ustawione na eksport tylko do ich odpowiednich platform.

Oficjalne rozszerzenia innych firm

Starsze wersje GameMaker miały wbudowane wiele funkcji stron trzecich, ale było to nieefektywne i trudne do utrzymania, ponieważ jeśli jakaś konkretna funkcja została zmieniona przez stronę trzecią, to całe GameMaker IDE i Runtime musiałyby zostać zaktualizowane. Tak więc, obecne wersje przyjęły użycie pakietów extensions, aby dodać obsługę innych firm do swoich gier. Oznacza to, że możesz dodać tylko te dodatkowe funkcje, których potrzebujesz, a nawet możesz je dostosować i edytować do swoich specyficznych potrzeb.

Większość z tych extensions zostanie pobrana i zainstalowana dla Ciebie, gdy wybierzesz konkretne Opcje Gry (na przykład Facebook lub Google Mobile Ads), ale możesz znaleźć więcej informacji na ich temat w odpowiedniej sekcji dla każdej platformy w Bazie Wiedzy GameMaker:

  • Platformy i konfiguracjaGameMakera

Samą extensions można znaleźć na stronie Marketplace:

  • Oficjalny Marketplace Aktywa

Należy pamiętać, że te extensions są dostarczane z demem i plikiem pomocy (w formacie *.pdf ), które wyjaśniają, jak z nich korzystać, i jako takie ich funkcje nie są omówione w tym podręczniku.
