JSDoc Script Uwagi

Jeśli chcesz, aby Twoje funkcje miały uzupełnianie kodu i pokazywały wymagane argumenty w określony sposób w edytorze kodu, to musisz dodać kilka komentarzy w stylu JSDoc. Komentarze te są używane do informowania funkcji autouzupełniania, jak funkcja powinna być używana i wypełniana w edytorze skryptów.

Format typowego nagłówka funkcji to nazwa funkcji, opis funkcji, a następnie lista różnych argumentów (parametrów), które funkcja przyjmuje, upewniając się, że każda linia zaczyna się od potrójnego odwrotnego ukośnika "///", ponieważ mówi to GameMakerowi, aby przetworzyć komentarz jako styl JSDoc. Możesz alternatywnie zawinąć swoje komentarze JSDoc w /**... */, przy czym /** jest w osobnej linii.

Samym komentarzom należy nadać identyfikator (poprzedzony znakiem "@") oraz treść, a dostępne identyfikatory są następujące:

Identyfikator Treść Składnia
@function / @func Pełna nazwa funkcji wraz z argumentami @func my_func(args)
@description / @desc Ogólny opis tego, co robi funkcja @desc <Description goes>
@param / @parameter / @arg /. @argument Informacja o argumentach, z opcjonalnym type @param {type} name <Parameter description>
@return / @returns Rodzaj danych, które zwraca funkcja @return {type} <Return description>
Tylkopióra
@pure Oznacza funkcję jako czystą @pure
@ignore Ukrywa funkcję z autocomplete Feathera @ignore
@deprecated Oznacza funkcję jako przestarzałą @deprecated
@context / @self Ustawia kontekst dla funkcji, który Feather auto-complete używa do zapewnienia informacji kontekstowych. Może być object lub konstruktorem. @self <object>

Jeśli nie określiłeś typu dla swoich argumentów lub wartości zwracanej, a używasz Feather, przyjmie on typ danych dla nich automatycznie, na podstawie ciała twojej funkcji.

Zobacz informacje o typach danych dla @param i @return na Feather Data Types.

Przykłady

Aby uzyskać pomysł, jak to będzie działać podczas pisania własnych funkcji, weźmy ten podstawowy przykład:

function is_same_object(_id, _obj)
{
    if (_id.object_index == _obj)
    {
        return true;
    }
    else return false;
}

Wszystko, co robi ten script, to sprawdzenie, czy instancja ma ten sam object_index co dany object i zostałaby nazwana po prostu jako:

if is_same_object(id, obj_Player)
{
    instance_destroy()
}

Jednak wpisanie tego do edytora kodu pokaże ci bezpośrednio nazwy zmiennych argumentów (_id i _obj), co w większości przypadków nie jest zbyt opisowe. Możesz użyć JSDoc, aby zdefiniować niestandardowe nazwy argumentów i opisy, wraz z informacjami dla funkcji, jak również:

/// @function                is_same_object(inst_id, object_id)
/// @description             Check if the given instance belongs to the given object.
/// @param {Id.Instance}     inst_id    The unique instance ID value of the instance to check.
/// @param {Asset.GMObject}  object_id  The object index to be checked against.
/// @return {Bool}

function is_same_object(_inst_id, _object_id)
{
    return _inst_id.object_index == _object_id;
}

Na powyższym obrazku, górna część pokazuje funkcję w autouzupełnianiu, a dolna część pokazuje jak działa helper argumentów na dole. @param Zauważ, że zarówno opcjonalna część "type", jak i obowiązkowa "description" nie są domyślnie wyświetlane w kodzie IDE complete i aby je zobaczyć, musisz aktywować opcje w Preferencjach GML.

Podczas korzystania z Feather, po najechaniu myszką na daną funkcję zobaczysz szczegółowe informacje na jej temat:

Możesz zawinąć nazwę argumentu w nawiasach [], aby wskazać, że jest on opcjonalny. Edytor kodu będzie wtedy oczekiwał dowolnej liczby argumentów pomiędzy minimalną wymaganą liczbą argumentów a całkowitą liczbą argumentów.

Na przykład, zobacz następującą funkcję:

/// @function    animate_position(end_x, end_y, start_x, start_y)
/// @desc        Animates the instance to ending point, from optional starting point
/// @arg end_x
/// @arg end_y
/// @arg [start_x]
/// @arg [start_y]

function animate_position (x1, y1, x2, y2)
{
    // Function code
}

Argumenty start_x i start_y są oznaczone jako opcjonalne, co oznacza, że edytor kodu będzie teraz oczekiwał od 2 do 4 argumentów, jak widać w komunikacie ostrzegawczym:

UWAGA Otrzymasz to samo zachowanie, jeśli użyjesz opcjonalnych argumentów w deklaracji funkcji. Zobacz funkcje skryptowe, aby uzyskać więcej informacji.
