Zmienne obiektowe

Po kliknięciu przycisku Zmienne zostanie otwarte okno Zmienne obiektu , które można wykorzystać do wygenerowania dowolnej liczby zmiennych przed zdarzeniem Utwórz obiekt:

W tym oknie można kliknąć przycisk Dodaj , aby dodać nową zmienną do listy. Następnie można nadać nazwę zmiennej, a także ustawić jej typ i dostosować właściwości:

Nazwa zmiennej musi zaczynać się od litery i może zawierać tylko litery, cyfry i symbol podkreślenia "_" o maksymalnej długości 64 symboli. o maksymalnej długości 64 symboli. Tak więc poprawne zmienne to takie rzeczy, jak fish, foo_bar, num1, a niepoprawne zmienne to 6fish, foo bar, lub *num. Po podaniu nazwy zmiennej musisz ustawić jej typ, który może być dowolny:Warto zauważyć, że ta funkcja jest szczególnie przydatna podczas pracy z instancjami typu rodzic/dziecko - ponieważ możesz mieć jednostkę nadrzędną object z zestawem zdefiniowanych zmiennych, a następnie po prostu modyfikować je w jednostkach podrzędnych object (więcej szczegółów znajdziesz w rozdziale Zmienne obiektu i obiekty podrzędne poniżej) - oraz podczas dodawania instancji w Edytorze pomieszczeń (zobacz rozdział Warstwy > Warstwa instancji), ponieważ możesz również modyfikować te zmienne dla poszczególnych instancji które zostały umieszczone na stronie room.

Poniżej wymieniono różne typy zmiennych, które można utworzyć:

  • Liczbarzeczywista: Liczba rzeczywista to każda liczba, która nie jest liczbą całkowitą i może być dodatnia lub ujemna. Tak więc 124.5, 45639.566546456, 0.9, -45.5 itd... to przykłady liczb rzeczywistych. Wszystkie liczby rzeczywiste są zapisywane jako 64-bitowe wartości zmiennoprzecinkowe (podwójne). Typy danych rzeczywistych (i całkowitych) mają opcję ustawiania zakresu wartości, do której można przejść, klikając przycisk Opcje :Po zaznaczeniu tej opcji można wprowadzić wartość początkową i końcową, a zamiast stałej wartości wyświetlanej dla zmiennej w oknie Object Zmienne, pojawi się suwak, który jest przyporządkowany do tych wartości:
  • Liczbacałkowita: Liczba całkowita to liczba całkowita, która może być dodatnia lub ujemna, na przykład 30004, 19, 0, -300. Zauważ, że możesz ustawić zakres wartości dla wyjściowej liczby całkowitej, jak wyjaśniono powyżej dla liczb rzeczywistych.
  • String: string to wszystko, co zostało ujęte w podwójne cudzysłowy, na przykład "fish", "Hello World" lub "12345".
  • Boolean: Boolean to wartość, która jest albo prawdziwa, albo fałszywa. W oknie Zmienne edytora Object jest to po prostu pole wyboru, które zaznacza się, gdy wartość jest prawdziwa, a odznacza, gdy jest fałszywa.
  • Wyrażenie: Wyrażenie to wyrażenie matematyczne, które może zawierać zwykłe liczby, zmienne, funkcje strings lub GML oraz jeden lub więcej operatorów. Na przykład sqrt(85 * 6) + 5.5 jest wyrażeniem.
  • Zasób: Zasób to po prostu dowolny z zasobów zdefiniowanych w przeglądarce Asset. Po wybraniu tego typu zasobu można kliknąć przycisk Otwórz Asset Explorer , aby otworzyć Asset Explorer i wybrać żądany zasób.
  • Lista: Wybranie typu listy oznacza, że można utworzyć listę wartości (mogą to być strings, liczby rzeczywiste, wyrażenia itd...), z których można wybrać jedną lub więcej do zwrócenia przez zmienną. Aby zdefiniować elementy, które znajdą się na liście, należy je najpierw dodać do listy Options (kliknij przycisk ):
    Dzięki temu w oknie głównym będzie można wybrać z menu rozwijanego pozycję jako wartość domyślną dla zmiennej:
    Zwróć uwagę, że jeśli w opcjach listy zaznaczono opcję Multi-select, zmienna stanie się tablicą wszystkich wybranych opcji:
  • Kolor: Typ koloru służy do definiowania wartości koloru, która ma być przechowywana w zmiennej. Można wprowadzić liczbę rzeczywistą (od 0 do 16777216, wartość alfa będzie wynosić 255) lub wartość heksadecymalną (w formacie $RRGGBBAA), albo można dwukrotnie kliknąć stronę na próbce koloru, aby otworzyć narzędzie Colour Picker (Próbnik kolorów) i tam zdefiniować kolor.

Zmienne obiektowe i obiekty-dzieci

Ważną cechą zmiennych Object jest to, że są one dziedziczone przez wszystkie obiekty potomne objects, które znajdują się w przeglądarce Asset (więcej informacji znajdziesz w rozdziale Obiekty nadrzędne), co oznacza, że możesz co oznacza, że użytkownik może nadpisać lub zmienić wszystkie lub niektóre z nich, jeśli zechce.

Gdy utworzysz dziecko object innego zasobu object, który ma zdefiniowane zmienne, pojawią się one w oknie Object Variables, jak poniżej:

Na górnym obrazie widać, że zmienna nadrzędna object ma cztery zmienne Object, a na dolnym obrazie te są również widoczne, tylko "wyszarzone" i mają ikonę Dziedziczone od rodzica , która wskazuje, że zostały one odziedziczone. Te odziedziczone zmienne można następnie edytować po kliknięciu przycisku Override Variable , dzięki czemu można dostosować suwak zakresu lub wartości, wybierać różne elementy z list itd. Należy pamiętać, że podczas edycji zmiennej nadrzędnej można zmienić tylko zdefiniowaną wartość, ale nie nazwę, nie można też zmienić opcji zmiennej. Zwróć też uwagę, że możesz dodawać nowe zmienne Object również do zmiennych potomnych objects (na powyższym obrazku widać, że zmienna "Typ znaku" jest nową zmienną Object tylko dla zmiennej potomnej object).

Jeśli edytowałeś dziedziczoną zmienną Object, możesz kliknąć przycisk Usuń , aby usunąć edycję, ale nie zmienną. Zmienna będzie nadal będzie dziedziczona od rodzica, ale ponownie będzie używała wartości domyślnej rodzica. Jeśli chcesz całkowicie usunąć zmienną, musisz to zrobić z poziomu okna nadrzędnego object.
