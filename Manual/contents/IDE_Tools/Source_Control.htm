Kontrola źródła

Zarządzanie kontrolą źródła (SCM) to nazwa nadana metodzie pracy z podwersjami lub kopiami zapasowymi projektów za pośrednictwem repozytorium i lokalnego źródła. Zasadniczo rozwiązanie SCM jest niezależnym pakietem oprogramowania, który kontroluje wszystkie aspekty utrzymania, zmiany i porównywania wersji projektu w trakcie pracy nad nim. Jest to szczególnie przydatne dla osób pracujących w zespole, które muszą mieć kontrolę nad tym, kto co robi, i nie muszą się martwić o utratę danych lub wprowadzanie zmian, które trzeba będzie później cofnąć, ale z tego potężnego i elastycznego systemu mogą korzystać także osoby indywidualne.

GameMaker korzysta z Git jako SCM, który musi być zainstalowany i skonfigurowany oddzielnie, zanim SCM będzie mógł być użyty w GameMaker. Więcej informacji na temat Git można znaleźć w Dokumentacji Git.

GameMaker wyświetli rozwijane menu na pasku menu dla Source Control, które zostanie wypełnione następującymi opcjami po aktywowaniu Source Control dla Twojego projektu (szczegóły w rozdziale Setting Up The Git Plugin powyżej):

Poniżej przedstawiamy w skrócie każdą z dostępnych opcji (większość z nich jest szczegółowo omówiona w powyższych sekcjach samouczka):

  • Create Project Repository: Ta opcja umożliwia utworzenie lokalnego repozytorium w tym samym katalogu, w którym znajdują się pliki projektu.
  • Sklonuj repozytorium: Ta opcja umożliwia sklonowanie repozytorium ze źródła do nowego miejsca docelowego.
  • Commit Changes: Dzięki tej opcji możesz etapować zmienione pliki w projekcie, a następnie zatwierdzić je w repozytorium.
  • Pchnij zmiany: Po wykonaniu commitu lub merge ta opcja służy do wypchnięcia zmian do repozytorium głównego.
  • Wyciągnij zmiany: Dzięki tej opcji można aktualizować lokalne repozytorium, pobierając zmienione pliki z repozytorium głównego.
  • Wyświetl historię: Otworzy to okno historii, w którym można przejrzeć całą historię wersji projektu i wybrać cofnięcie określonego pliku paths lub całych rewizji.
  • Show Conflicts (Pokaż konflikty): Spowoduje to otwarcie okna Konflikty i wyświetlenie listy wszystkich plików, które mogą być w konflikcie w projekcie, umożliwiając zajęcie się nimi w programie GameMaker IDE lub za pomocą specjalnego narzędzia Merge Tool.
