Polecenie throw runtime pozwala na generowanie własnych wyjątków, kończących grę i wyświetlających komunikat o błędzie:

throw (<expression>);

Użyte wyrażenie może być wartością, adresem string lub dowolnym innym typem danych, a następnie zostanie wygenerowany błąd wyjątku, który - domyślnie - zostanie wyświetlony na ekranie, a po zamknięciu komunikatu o błędzie gra zostanie zakończona. Na przykład wywołanie tego:

throw ("Hello World!");

Można jednak przejąć ten komunikat o błędzie i użyć własnego kodu obsługi, wywołując funkcję exception_unhandled_handler(). Ta funkcja runtime umożliwia dostarczenie własnego kodu obsługi method który będzie wywoływany za każdym razem, gdy w grze wystąpią nieobsługiwane wyjątki.
