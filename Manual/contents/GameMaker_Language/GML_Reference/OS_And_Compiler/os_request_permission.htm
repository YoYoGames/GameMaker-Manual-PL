os_check_permission

Za pomocą tej funkcji możesz zażądać określonego uprawnienia. Uprawnienie, o które chcesz poprosić, podajesz jako string w formacie "android.permission.<permission>", więc aby poprosić o uprawnienie RECORD_AUDIO (na przykład), wywołałbyś

os_request_permission("android.permission.RECORD_AUDIO");

Funkcja zwraca zdarzenie asynchroniczne systemu, w którym wbudowana mapa async_load DS będzie zawierała następujące pary klucz/wartość:

  • "type": Jest to typ zdarzenia, które zostało wywołane, a wartością będzie string "permission_request_result".
  • "<permission>": Będzie to zezwolenie żądane przez funkcję w postaci string, które będzie zawierać jedną z następujących stałych:
Stała Opis
os_permission_granted Oznacza to, że zezwolenie zostało udzielone
os_permission_denied Oznacza to, że pozwolenie nie zostało udzielone

Warto zauważyć, że poniższe uprawnienia są obsługiwane natywnie przez GameMakera, ale są uważane przez Google za "niebezpieczne" i jako takie muszą być wyraźnie wymagane (zauważ też, że niektóre uprawnienia mogą być wymagane za pomocą Opcji Gry Android bez potrzeby korzystania z tej funkcji):

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

Więcej informacji na temat uprawnień do aplikacji można znaleźć w Dokumentacji systemu Android.

WAŻNE! Ta funkcja jest przeznaczona wyłącznie dla systemu Android.

Składnia:

os_request_permission(permission)

Argument Typ Opis
permission String The permission to request (a string)

Zwroty:

N/A

Przykład:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

Powyższy kod sprawdza typ systemu operacyjnego i jeśli jest to Android, sprawdza uprawnienia i jeśli uprawnienie "INTERNET" nie zostało jeszcze przyznane, żąda go.
