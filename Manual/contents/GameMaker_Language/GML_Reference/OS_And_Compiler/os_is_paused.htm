os_is_paused

Podczas tworzenia gier na urządzenia mobilne trzeba mieć możliwość sprawdzenia, czy urządzenie jest nadal skoncentrowane na grze, czy nie. Zwłaszcza że gra na urządzeniu iOS, Android lub Windows Phone może zostać przerwana przez rozmowy telefoniczne i inne aplikacje. Za pomocą tej funkcji można sprawdzić, czy urządzenie jest wstrzymane, czy nie.

Warto zauważyć, że ta funkcja uruchomi na urządzeniach dowolny kod zawarty w bloku kodu, ale zaraz potem GameMaker zostanie wstrzymany, ponieważ urządzenie caches gra jako proces działający w tle. Jeśli więc masz coś takiego, jak w poniższym przykładzie, tworzona instancja nie pojawi się ani nie wykona żadnego ze swoich zdarzeń, dopóki gra nie zostanie ponownie uruchomiona z buforowanego stanu. Oznacza to, że lepiej jest, aby funkcja os_is_paused wywoływała alarm, a następnie w zdarzeniu alarmowym (które zostanie ustawione, gdy urządzenie zostanie wstrzymane, ale nie zostanie uruchomione, dopóki nie zostanie ponownie wstrzymane) tworzyła menu pauzy, uruchamiała ponownie muzykę itd.

Jeśli korzystasz z tej funkcji na platformach Windows, Ubuntu (Linux), MacOS lub HTML5, funkcja jest przełączana tylko na jeden krok do true, gdy gra traci fokus, a następnie powraca do false ponownie, dopóki gra nie odzyska fokusu (i ponownie go nie straci). Dzieje się tak dlatego, że na tych platformach nie ma strony cache aplikacje i gry będą nadal działać w tle.

UWAGA: Na urządzeniach przenośnych dźwięki i muzyka zostaną zatrzymane po wstrzymaniu os. Nie zostaną one ponownie uruchomione, chyba że użytkownik uruchomi je za pomocą odpowiedniej funkcji.

Składnia:

os_is_paused()

Zwroty:

Przykład:

if os_is_paused()
{
    if !instance_exists(obj_PauseMenu)
    {
        instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
    }
}

Powyższy kod sprawdza, czy gra została wstrzymana, a jeśli tak, to tworzy instancję object "obj_PauseMenu", jeśli żadna jeszcze nie istnieje.
