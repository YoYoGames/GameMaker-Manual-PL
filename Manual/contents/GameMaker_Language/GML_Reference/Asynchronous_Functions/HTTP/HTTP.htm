HTTP

W tym rozdziale wymieniono wszystkie różne funkcje asynchronicznego protokołu HTTP dostępne w programie GameMaker. Funkcje te będą generować zdarzenie Asynchroniczny HTTP we wszystkich instancjach, które je posiadają:

  • http_request
  • http_get
  • http_get_file
  • http_post_string

Należy pamiętać, że powyższe funkcje http_* mogą nie działać zgodnie z oczekiwaniami z powodu problemów z bezpieczeństwem domen. Oznacza to, że żądania kierowane do witryny server lub próby załadowania zasobów z innych domen są blokowane i mogą zwracać puste wyniki lub błędy 404. Jednym ze sposobów na obejście tego problemu jest zainstalowanie po stronie server języka PHP, który pozwala niektórym domenom na dostęp do server (jest to również sposób na ochronę zasobów i blokowanie servers, które nie znajdują się na liście PHP allow).

Poniżej znajduje się przykład kodu PHP, który można wykorzystać w tym celu:

$http_origin = $_SERVER['HTTP_ORIGIN'];
if ($http_origin == "http://127.0.0.1:51268")
{
    header("Access-Control-Allow-Origin: *");
}

W przypadku żądań ładowania obrazów, w których ważne jest określenie lub ustawienie ich typu pochodzenia krzyżowego, dostępne są następujące funkcje:

  • http_get_request_crossorigin
  • http_set_request_crossorigin

W większości przypadków korzystanie z tych funkcji nie powinno być konieczne, ale jeśli gra jest przechowywana na zabezpieczonej stronie server - gdzie niektóre strony assets mogą wymagać podstawowego uwierzytelnienia, aby uzyskać do nich dostęp i generują błędy zabezpieczeń podczas ładowania - konieczne może być ustawienie typu cross-origin dla żądań obrazu na "use-credentials" (w przeciwieństwie do domyślnego ustawienia "anonymous").
