push_cancel_local_notification

Ta funkcja anuluje podane powiadomienie push. Identyfikator powiadomienia do anulowania można uzyskać za pomocą funkcji push_get_first_local_notification() i push_get_next_local_notification() a funkcja zwróci true w przypadku powodzenia lub false w przeciwnym przypadku (np. jeśli powiadomienie nie istnieje lub zostało już wywołane).

UWAGA: Ta funkcja jest ograniczona do modułów docelowych iOS i Android.

Składnia:

push_cancel_local_notification(ID);

Argument Typ Opis
ID Identyfikator powiadomienia Push The ID of the notification to cancel.

Zwroty:

Boolean

Przykład :

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
    {
        push_cancel_local_notification(ntf);
    }
    ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

Powyższy kod tworzy stronę DS Map, a następnie żąda danych dla pierwszego dostępnego powiadomienia na stronie queue. Jeśli takie powiadomienie istnieje, mapa jest wypełniana, a następnie kod sprawdza klucz "data" pod kątem konkretnego adresu string. Jeśli string zostanie znalezione, powiadomienie jest anulowane i przechodzi do sprawdzania kolejnego powiadomienia, aż do sprawdzenia wszystkich w queue.
