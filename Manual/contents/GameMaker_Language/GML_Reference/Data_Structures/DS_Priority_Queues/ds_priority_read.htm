ds_priorytet_czytania

Za pomocą tej funkcji można odtworzyć zapisany priorytet DS (zapisany wcześniej jako string przy użyciu ds_priority_write()). Musisz najpierw utworzyć nowy priorytet DS, do którego wczytasz string, a jeśli priorytet DS już istnieje i ma zapisane informacje, zostaną one wyczyszczone przed odczytem. Funkcja ta ma kluczowe znaczenie przy tworzeniu mechanizmów zapisywania i wczytywania danych w grze.

Zwróć uwagę, że jeśli określony plik DS string został napisany w programie GameMaker: Studio 1.2.x runtime (lub starszym), należy podać opcjonalny argument "legacy", ustawiając go na true, ponieważ format string zmienił się po tej wersji.

Składnia:

ds_priority_read(id, str, [legacy]);

Argument Typ Opis
id Identyfikator priorytetu DS The id of the data structure to write.
str String The string to read from.
legacy Boolean OPTIONAL Można podać adres true lub false albo całkowicie go pominąć.

Zwroty:

N/A

Przykład:

p_queue = ds_priority_create();
ini_open("save.ini");
var str = ini_read_string("P_Queues", "0", "");
if (str != "")
{
    ds_priority_read(p_queue, str);
}
ini_close();

Powyższy kod tworzy priorytet queue i zapisuje jego indeks w zmiennej "p_queue". Następnie otwiera plik ini i odczytuje z niego plik string, sprawdzając, czy plik string nie jest najpierw zwracany jako pusty. Ta strona string jest następnie odczytywana do nowo utworzonego priorytetu DS.
