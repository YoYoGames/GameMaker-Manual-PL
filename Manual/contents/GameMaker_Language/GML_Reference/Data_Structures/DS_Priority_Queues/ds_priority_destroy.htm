ds_priorytet_zniszczenia

Ta funkcja usuwa z pamięci podaną strukturę danych o priorytecie queue, zwalniając zasoby, z których korzystała, i usuwając wszystkie wartości, które zawierała. Funkcji tej należy używać zawsze po zakończeniu korzystania z priorytetu DS queue, aby zapobiec wyciekom pamięci, które mogą spowolnić i zawiesić grę.

WAŻNE! Kiedy tworzysz strukturę danych, wartość indeksu identyfikująca ją jest wartością całkowitą zaczynającą się od 0. Oznacza to, że struktury danych różnych typów mogą mieć tę samą wartość indeksu, więc jeśli masz wątpliwości, powinieneś użyć funkcji ds_exists() przed uzyskaniem do nich dostępu. Należy również pamiętać, że indeksy są używane ponownie, więc wartość indeksu zniszczonej struktury danych może zostać użyta przez nowo utworzoną strukturę. Dlatego zalecamy, aby po zniszczeniu zawsze ustawiać zmienną, która przechowywała indeks DS, na -1.

Składnia:

ds_priority_destroy(id);

ArgumentTyp Opis
id The id of the data structure to remove.

Zwroty:

Przykład:

if (lives == 0)
{
    ds_priority_destroy(AI_queue);
    AI_queue = -1;
    room_goto(rm_Menu);
}

Powyższy kod sprawdzi wartość wbudowanej zmiennej globalnej "lives" i jeśli będzie ona równa 0, zniszczy priorytet DS queue zaindeksowany w zmiennej "AI_queue", a następnie zmieni adres rooms.

Powrót: Kolejki priorytetowe DS
Następny: ds_priority_clear
