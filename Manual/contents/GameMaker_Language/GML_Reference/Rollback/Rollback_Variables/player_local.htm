gracz_lokalny

Kiedy używasz rollback_define_player() do zdefiniowania gracza object, system automatycznie tworzy instancję tego object dla każdego gracza. Każda instancja tego object otrzymuje zmienną player_local, która mówi, czy ta instancja gracza jest lokalna czy nie.

Jeśli nie używasz rollback_define_player(), to ta zmienna nie zostanie utworzona dla żadnego objects.

Możesz użyć tej zmiennej do pokazania, który gracz jest lokalny. Nie zaleca się używania tej zmiennej w sposób, który wpływa na twoją logikę gry, która powinna być taka sama dla graczy lokalnych i zdalnych.

Składnia:

player_local;

Zwroty:

Boolean

Przykład:

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

Powyższy kod jest uruchamiany w zdarzeniu Draw i sprawdza, czy instancja gracza jest lokalna. W takim przypadku rysuje strzałkę sprite nad instancją, informując gracza, którą instancją steruje.

Powrót: Zmienne cofania
Następny: player_name
