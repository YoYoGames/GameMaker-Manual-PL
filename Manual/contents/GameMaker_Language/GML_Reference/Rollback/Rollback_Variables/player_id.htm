identyfikator_gracza

Gdy użyjesz rollback_define_player() do zdefiniowania gracza object, system automatycznie tworzy instancję tej strony object dla każdego gracza. Każda instancja tej strony object otrzymuje zmienną player_id, która przechowuje identyfikator zawodnika.

Identyfikator ten zaczyna się od 0, czyli początkowo jest to sam host. Wszyscy gracze od 1 wzwyż są "współgraczami", którzy dołączyli do gry.

Jeśli nie używasz rollback_define_player(), ta zmienna nie zostanie utworzona dla żadnego objects. W takim przypadku instancje graczy i ich identyfikatory muszą być zarządzane ręcznie, jak wyjaśniono w rozdziale Definiowanie obiektu gracza.

Składnia:

player_id;

Zwroty:

Real

Przykład:

// Create event
if (player_id == 0)
{
    x = 300;
}
else if (player_id == 1)
{
    x = 500;
}

Powyższy kod zmienia pozycję X instancji gracza na podstawie jej identyfikatora.

Tego identyfikatora można także użyć jako indeksu tablicy, na przykład do przechowywania wyników wszystkich zawodników:

// Step event
var _coin = instance_place(x, y, obj_coin);
if (_coin != noone)
{
    obj_game.scores[player_id]++;
    instance_destroy(_coin);
}

// Draw event
var _my_score = obj_game.scores[player_id];
draw_text(x, y, _my_score);

Powyższy kod zdarzenia Step zwiększa wynik gracza w tablicy w obj_game, gdy zderzy się on z monetą. Ta instancja monety jest następnie niszczona.

Powyższy kod zdarzenia Draw pobiera wynik gracza z tej samej tablicy i rysuje jego wartość.
