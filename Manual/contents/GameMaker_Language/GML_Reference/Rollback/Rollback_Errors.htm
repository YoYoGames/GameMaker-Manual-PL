Błędy cofania

Podczas pracy z systemem Rollback mogą wystąpić pewne błędy. Błędy te są błędami runnera i dlatego są pokazywane podczas działania gry:

Poniższa tabela zawiera listę możliwych błędów, które można otrzymać podczas pracy z systemem Rollback wraz z opisem. Podaje ona również rozwiązanie dla każdego konkretnego komunikatu o błędzie.

Błąd Opis
Nie udało się uzyskać warstwy do tworzenia graczy. Powinieneś przekazać nazwę warstwy do funkcji rollback_define_player lub utworzyć warstwę Instances. Jeśli gracz object został zdefiniowany za pomocą funkcji rollback_define_player, system rollback spróbuje stworzyć gracza object na warstwie, której nazwę można opcjonalnie przekazać jako drugi parametr (lub na warstwie o nazwie "Instances", jeśli nie została zdefiniowana). Jeśli nie uda się znaleźć warstwy po jej nazwie, zostanie wywalony ten błąd. Rozwiązaniem jest albo jawne ustawienie nazwy warstwy, na której mają zostać utworzeni gracze, albo utworzenie warstwy o nazwie "Instances", jeśli nie ustawimy jawnie nazwy warstwy.
Gracz objects musi być zarządzany podczas uruchamiania gier wieloosobowych. Jeśli gracz object został zdefiniowany za pomocą funkcji rollback_define_player, a gracz object nie jest oznaczony jako zarządzany, to ten błąd zostanie odpalony. Rozwiązaniem jest oznaczenie gracza object jako zarządzanego w edytorze obiektów.
Failed to get user info for player_id N. Jeśli system cofania nie był w stanie uzyskać informacji z server na temat gracza, ten błąd zostanie odpalony. Sprawdź w konsoli deweloperskiej, czy jakieś żądania zostały zablokowane.
Nie można było wywołać rollback_chat przed pełnym połączeniem z grą. Spróbuj poczekać na zdarzenie rollback_game_info. Funkcja czatu nie jest dostępna, dopóki gracz nie dołączy do room. Rozwiązanie: Wyłącz czat do czasu otrzymania zdarzenia rollback_game_info (Patrz Zdarzenia Rollback).
rollback_use_manual_start musi być wywołany przed rollback_create_game i rollback_join_game. Błędy te są uruchamiane, jeśli wywołanie funkcji narusza wewnętrzne ograniczenia maszyny stanu. Komunikat o błędzie sugeruje rozwiązanie.
rollback_use_player_prefs musi być wywołany przed rollback_create_game i rollback_join_game.
rollback_use_player_prefs musi być wywołany przed rollback_set_player_prefs.
rollback_set_player_prefs musi być wywołany przed rollback_start_game.
rollback_define_player musi być wywołany przed rollback_create_game i rollback_join_game.
rollback_define_input musi być wywołany przed rollback_create_game i rollback_join_game.
rollback_define_mock_input musi być wywołany przed rollback_create_game i rollback_join_game.
Nie czytaj informacji o graczu przed rozpoczęciem gry. Wywołanie do rollback_get_info().
Nie czytaj danych wejściowych gracza przed rozpoczęciem gry. Wywołanie do rollback_get_input().
rollback_define_input_frame_delay musi być wywołany przed rollback_create_game i rollback_join_game
rollback_define_input_frame_delay musi być wywołany przed rollback_create_game i rollback_join_game.
rollback_set_player_prefs nie udało się ustawić preferencji Strona server odrzuciła prefs gracza. Zazwyczaj dzieje się tak w następujących przypadkach: 1. Jeden użytkownik próbuje zresetować ustawienia gracza, podczas gdy drugi użytkownik już rozpoczął grę. W tym przypadku gra powinna używać ostatnio ustawionych preferencji użytkownika, gwarantujemy, że każdy gracz powinien ustawić jakieś preferencje użytkownika przed rozpoczęciem gry. 2. Serializowany rozmiar prefów gracza przekracza maksymalny rozmiar prefów gracza (obecnie 800 bajtów).
rollback_create_game, liczba graczy ustawiona na N, ale musi być w zakresie 1 do 4 Funkcja rollback_create_game była wywoływana z ustawioną nieprawidłową liczbą graczy. Rozwiązaniem jest naprawienie pierwszego parametru rollback_create_game, aby był w zakresie od 1 do 4.
rollback_create_game nie udało się ustawić domyślnych preferencji gracza. Zobacz opis dla "rollback_set_player_prefs failed to set the preferences"
Multiplayer rollback jest obsługiwany tylko w celu operagx. Wywołana jest funkcja rollback_create_game, a cel nie jest ustawiony na OperaGX. Obecnie multiplayer jest obsługiwany tylko na platformie OperaGX. Rozwiązaniem jest ustawienie celu na OperaGX.
Nie udało się stworzyć gry (nieznany typ sesji). To jest wewnętrzny błąd systemu rollback. Zgłoś problem do zespołu GameMakera.
Could not find player_id on this instance, musisz przekazać go do rollback_get_input(). Funkcjarollback_get_input może przyjąć jako parametr id gracza. Jeśli nie podano żadnych parametrów, funkcja pobierze pole player_id gracza object aby uzyskać dane wejściowe. Jeśli parametr player_id nie jest ustawiony, to zostanie wywołany błąd. Jeśli gracz object został utworzony za pomocą rollback_define_player powinien mieć ustawiony player_id. Rozwiązanie: 1. Przekaż jawnie parametr player id do funkcji 2. Utwórz gracza za pomocą rollback_define_player.
Info player_id N jest poza zasięgiem. rollback_get_info jest wywoływany z niewłaściwym player_id (lub gracz object ma złą wartość w polu player_id). Rozwiązanie: Popraw parametr player_id, aby był w zakresie od 1 do 4.
Nie udało się znaleźć instancji <id> strony object <id> (<nazwa>), do której odwołują się dane rollback. Wewnętrzny błąd systemu rollback. Zgłoś problemdo zespołu GameMakera.
Nie ma pojęcia jak stworzyć OBJECT_KIND=<kind>.
RefID <id> nie ma object (źle sformułowane dane wejściowe?).
Invalid ReferenceKind <id> (malformed input?).
Mapowanie instancji jest włączone, ale próbowaliśmy deserializować indeks spoza zakresu. <id> nie jest w [0, <id>].
Invalid RValue w deserializacji.
Próba wskrzeszenia instancji, która została usunięta!
Can't serialize RValue with kind=<id> (<name>).
Pusta definicja wejścia. rollback_define_input został wywołany z pustym wejściem.
Definicja wejścia zawiera stałą <kod klucza> dwa razy, co jest niedozwolone. Kod klucza może być przypisany tylko do jednego wejścia. Sprawdź, czy jakiś kod klucza jest ustawiony więcej niż raz w definicji wejścia.
Definicja wejściowa z etykietą <label> nie jest wartością rzeczywistą ani tablicą wartości rzeczywistych. Wszystkie wartości opisujące wartości wejściowe powinny mieć typ Real.
Pusta definicja wejściowa mock dla player_id N. Zobacz "Definicja pustego wejścia".
Definicja Mock zawiera wartość nierzeczywistą. Zobacz "Definicja wejściowa z etykietą <label> nie jest wartością rzeczywistą ani tablicą wartości rzeczywistych."
