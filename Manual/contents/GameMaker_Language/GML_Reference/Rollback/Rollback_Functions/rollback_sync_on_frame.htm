rollback_sync_on_frame

Funkcja ta sprawdza, czy wszyscy gracze są zsynchronizowani na bieżącej klatce, co oznacza, że ich stany nie są oparte na przewidywaniach, ale na rzeczywistych danych otrzymanych od nich. W takim przypadku zwraca wartość true.

Jeśli gracze nie są zsynchronizowani na bieżącej klatce, zwraca false i próbuje zsynchronizować wszystkich graczy.

Próba synchronizacji graczy za pomocą tej funkcji może spowodować zamrożenie. Musisz zapewnić, że ta funkcja nie jest stale wywoływana w zdarzeniu Step.

Zamiast tego użyj go jako przełącznika. Zobacz przykład na dole strony.

Zmienna rollback_confirmed_frame jest aktualizowana automatycznie, gdy wszyscy gracze są zsynchronizowani.

Składnia:

rollback_sync_on_frame();

Zwroty:

Boolean

Przykład:

if (obj_game.winning_player == -1 && points >= obj_game.points_required_for_win)
{
    if (rollback_sync_on_frame())
    {
        obj_game.winning_player = player_id;
    }
}

Ten kod działa w zdarzeniu Step obiektu gracza.

Sprawdza, czy nie ma wygrywającego gracza i czy obecny gracz ma punkty wymagane do wygranej.

Jeśli oba są prawdziwe, oznacza to, że gracz powinien wygrać. Jednak zanim to zrobi, sprawdza, czy wszyscy gracze są zsynchronizowani, aby upewnić się, że stan jest poprawny.

Jeśli wszyscy gracze są zsynchronizowani, ustawia zmienną winning_player. Powoduje to, że pierwszy warunek staje się fałszywy, zapewniając, że rollback_sync_on_frame() nie zostanie ponownie wywołany.

Oznacza to, że funkcja jest używana jako przełącznik, zamiast działać nieprzerwanie w każdej klatce.
