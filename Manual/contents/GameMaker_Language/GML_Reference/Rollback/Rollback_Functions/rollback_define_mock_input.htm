rollback_define_mock_input

Tej funkcji można użyć podczas testu synchronizacji w celu przypisania tymczasowego "wyśmiewanego" wejścia do określonego odtwarzacza. Zastępuje ona elementy sterujące zdefiniowane dla tego odtwarzacza oraz wszelkie losowe dane wejściowe z testu synchronizacji.

Informacje na temat jego użycia można znaleźć w temacie Wejście próbne.

Składnia:

rollback_define_mock_input(player_id, input_struct);

Argument Typ Opis
player_id Real The ID of the player to assign mock input to.
input_struct Struktura A struct containing input names as variables, and their assigned inputs as values.

Zwroty:

N/A

Przykład:

rollback_define_input({
    fire: mb_left,
    interact: vk_space,
    left: ord("A"),
    right: ord("D")
});

rollback_define_mock_input(1, {
    fire: vk_control,
    interact: vk_shift,
    left: ord("J"),
    right: ord("L")
});

Powyższy kod najpierw definiuje wejścia dla graczy. Następnie definiuje kpiące wejście dla gracza 1, zachowując te same nazwy wejść, ale przypisując im różne wejścia. W ten sposób dwie osoby mogą lokalnie grać w grę przy użyciu klawiatury.
