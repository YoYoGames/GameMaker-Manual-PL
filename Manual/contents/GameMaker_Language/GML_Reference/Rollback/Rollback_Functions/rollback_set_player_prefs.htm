rollback_set_player_prefs

Po włączeniu preferencji, ta funkcja jest używana do ustawienia preferencji dla lokalnego gracza. Oznacza to, że funkcja musi polegać na lokalnych danych wejściowych, a nie na danych wejściowych Rollback.

Preferencje mogą być dowolnego typu. Jeśli ustawione są domyślne preferencje, upewnij się, że używasz tego samego typu danych i struktury.

Kiedy któryś z graczy ustawia swoje preferencje, wszyscy w grze otrzymują Rollback Event.

Jeśli jest używana, funkcja ta musi być uruchomiona przed funkcją join/create.

Proszę przeczytać: Preferencje gracza

Składnia:

rollback_set_player_prefs(preferences);

Argument Typ Opis
preferences Każdy The preferences to set for the local player.

Zwroty:

N/A

Przykład:

rollback_set_player_prefs({
character: obj_customizer.character_selected,
color: obj_customizer.color_selected
});

Powyższy kod ustawia preferencje dla gracza, odczytując wartości z "customizera" object, który byłby niezarządzany (lokalny).
