wycofać się z gry

Funkcja ta opuszcza bieżącą sesję multiplayer, umożliwiając graczowi dołączenie lub ponowne utworzenie nowej gry.

Kiedy gospodarz room odchodzi, własność tego room jest przekazywana innemu graczowi, więc gra może być kontynuowana. Gra kończy się dopiero wtedy, gdy wszyscy gracze odejdą.

Unikanie błędów

Po opuszczeniu przez gracza sesji wieloosobowej strona GameMaker kontynuuje wykonywanie całego kodu gry, jednak powraca do stanu "przedstartowego", w którym nie można tworzyć ani modyfikować zarządzanych instancji.

Oznacza to, że musisz ręcznie zniszczyć instancje graczy przy opuszczaniu gry i wstrzymać każdą inną logikę gry za pomocą rollback_game_running w warunku, aby uniknąć błędów.

Zobacz: Stan przed rozpoczęciem cofania

Zdarzenie wyzwolone

Kiedy gracz opuszcza grę, w której jesteś obecny, wywołane zostaje zdarzenie Rollback Event. Więcej informacji znajdziesz w dziale Rollback Events.

Składnia:

rollback_leave_game();

Zwroty:

N/A

Przykład:

if (keyboard_check_pressed(vk_escape))
{
    rollback_leave_game();
}

Powyższy kod sprawia, że gracz opuszcza grę po wciśnięciu klawisza Escape.
