Preferencje cofania

Możesz pozwolić każdemu graczowi wybrać postać lub dostosować jej właściwości. Takie właściwości mogą być ustawione jako "preferencje gracza".

Jak korzystać z preferencji?

Najpierw włącz preferencje, wywołując rollback_use_player_prefs() / Use Player Prefs. Dzięki tej funkcji możesz opcjonalnie ustawić domyślne preferencje dla wszystkich graczy.

Po wywołaniu tej funkcji, "auto-start" jest wyłączony, co oznacza, że gra nie rozpocznie się nawet wtedy, gdy wszyscy gracze dołączą, i będziesz musiał rozpocząć ją ręcznie za pomocą rollback_start_game() / Start Game.

Przed rozpoczęciem gry, możesz ustawić preferencje dla gracza, wywołując rollback_set_player_prefs() / Set Player Prefs na kliencie tego gracza. Mogą one być później odczytane przez rollback_get_player_prefs() / Get Player Prefs, lub zmiennej player_prefs w instancji gracza.

Zdarzenie cofnięcia

Kiedy gracz zmienia swoje preferencje, wszyscy w grze otrzymują rollback_player_prefs Rollback Event. Gracz musi być całkowicie połączony, aby zdarzenie zostało wysłane.

Jeśli preferencje zostały ustawione przed połączeniem, to po zakończeniu połączenia do wszystkich graczy wysyłane są ostatnie preferencje ustawione przed połączeniem.

Warunek początkowy

Jeśli włączyłeś preferencje, a nie ma ustawionych domyślnych preferencji, to gra nie rozpocznie się, jeśli nie zostaną ustawione preferencje dla wszystkich graczy.

W takim przypadku możesz albo wywołać rollback_start_game() wielokrotnie w alarmie, aby spróbować uruchomić grę, lub wywołać go za każdym razem, gdy zostanie odebrane zdarzenie rollback_player_prefs Rollback Event.

Jakich typów mogę używać?

Preferencja może być dowolnego typu, np. liczba, string, tablica itp. ale zaleca się używanie struct, który jest najprostszym sposobem przechowywania kolekcji zmiennych.
