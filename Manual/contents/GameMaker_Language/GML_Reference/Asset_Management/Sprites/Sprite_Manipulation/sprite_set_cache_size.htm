sprite_set_cache_size

Podczas mieszania obrazów HTML5 nie może robić tego dynamicznie w taki sam sposób, w jaki robiłby to program wykonywalny. Dlatego GameMaker musi tymczasowo zapisać kopię mieszanego obrazka, a następnie ją wczytać. Ta funkcja określa, ile kopii mieszania danej strony sprite może zostać zbuforowanych, zanim stare kopie zostaną nadpisane. Domyślnie jest to 4. Wartość ta jest stosowana do wszystkich podobrazów strony sprite.

Składnia:

sprite_set_cache_size(ind, max);

ArgumentTyp Opis
ind The index of the sprite to change the cache size of.
max The maximum number of cached copies of the sprite that can be stored.

Zwraca

Przykład:

sprite_set_cache_size(sprite0, 2);

Spowoduje to ustawienie adresu sprite cache wszystkich podobrazów sprite0 na 2 kopie.
