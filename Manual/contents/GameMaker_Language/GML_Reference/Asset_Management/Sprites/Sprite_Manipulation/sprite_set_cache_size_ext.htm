sprite_set_cache_size_ext

Podczas mieszania obrazów HTML5 nie może robić tego dynamicznie w taki sam sposób, w jaki robiłby to program wykonywalny. Dlatego GameMaker musi tymczasowo zapisać kopię mieszanego obrazu i wczytać ją. Ta funkcja określa, ile kopii mieszanki dla danego sprite może zostać zbuforowanych, zanim stare zostaną nadpisane. Domyślnie jest to 4. Wartość ta jest stosowana do pojedynczego podobrazu danej strony sprite.

Składnia:

sprite_set_cache_size_ext(ind, index, max);

Argument Typ Opis
ind Aktywa Sprite The index of the sprite to change the cache size of.
index Real The sub-image of ind to change the cache size of.
max Real The maximum number of cached copies of the sprite that can be stored.

Zwraca

N/A

Przykład:

sprite_set_cache_size_ext(sprite0, 0, 2);

Spowoduje to ustawienie adresu sprite cache pierwszego podobrazu sprite0 na 2 kopie.
