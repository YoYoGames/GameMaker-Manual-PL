sprite_merge

Ta funkcja połączy obraz sprite o indeksie w argumencie 1 ("ind2") z obrazem o indeksie w argumencie 0 ("ind1"). Same obrazy NIE są łączone ze sobą, ale raczej indeksy obrazów są łączone, a podobrazy z sprite "ind2" są dołączane do tych z sprite "ind1", czyli dodawane na końcu. Zwróć uwagę, że jeśli obrazy sprites mają różne rozmiary, to dołączone obrazy sprites są rozciągane, aby dopasować je do rozmiaru obrazu dla "ind1".

Zmiana ta jest trwała i od momentu użycia tej funkcji do momentu zamknięcia gry lub usunięcia strony sprite, strona sprite, z którą jest łączona, zostanie zmieniona, jednak strona sprite, z którą jest łączona, pozostanie bez zmian.

UWAGA: Ani obraz asset, który jest łączony, ani obraz asset, który ma być łączony, nie mogą być jednym z oryginalnych zasobów gry. Musisz użyć sprite_duplicate() first aby utworzyć dwa nowe zasoby (jeśli scalasz dołączoną grę assets), a następnie je scalić.

UWAGA: Ta funkcja jest przydatna tylko w przypadku map bitowych sprites i nie będzie działać z plikami SWF lub JSON (Spine) sprites.

Składnia:

sprite_merge(ind1, ind2);

ArgumentTyp Opis
ind1 The index of the sprite to merge.
ind2 The index of the sprite that ind1 is to be merged with.

Zwraca

Przykład:

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]);

Powyższy kod ładuje stronę sprite do zmiennej lokalnej, a następnie łączy ją z inną sprite, która została zduplikowana. Na koniec połączona strona sprite jest przypisywana do instancji sprite_index, a załadowana strona sprite jest usuwana aby zwolnić wykorzystywaną przez nią pamięć. Zauważ, że w pewnym momencie po wykonaniu tej operacji, np. w zdarzeniach Destroy lub Room End, będziesz musiał także użyć polecenia sprite_delete() aby usunąć połączonego sprite'a z pamięci.
