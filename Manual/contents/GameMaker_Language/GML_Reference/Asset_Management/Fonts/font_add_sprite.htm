czcionka_add_sprite

Za pomocą tej funkcji można użyć "paska sprite" do utworzenia nowego adresu font asset, gdzie każdy podobraz będzie indywidualnym symbolem lub literą. Indeks jest numerem mapy UTF8 dla pierwszego znaku, którego należy użyć, a zakresy, których należy użyć, są takie same jak w przypadku zwykłego font, zaczynając od "!", ale należy pamiętać, że należy uwzględnić tylko to, co jest potrzebne, i że należy użyć funkcji ord() aby uzyskać prawidłową wartość UTF8 dla początkowej litery. Utworzona strona sprite font będzie miała pod-obrazy ustawione na poszczególne glify z font, gdzie "pierwszy" glif będzie miał image_index równy 0. Możesz zdefiniować "spację" jako dowolny znak, na przykład pojedynczą linię o rozmiarze, jaki chcesz, aby miała spacja, a wszystkie spacje w tekście będą renderowane z tą szerokością (ale wybrany obraz nigdy nie będzie renderowany), lub jeśli nie podasz adresu sprite dla spacji, zostanie użyta szerokość najszerszego znaku w sprite font.

Można także ustawić odstępy między literami font, aby były proporcjonalne (true) lub nie (false), przy czym odstępy proporcjonalne font są zależne od rzeczywistej szerokości liter (na przykład litera "i" zajmuje mniej miejsca room niż litera "w"), natomiast odstępy nieproporcjonalne font są zależne od szerokości podobrazia, podobnie jak odstępy jednoprzestrzenne font. Na koniec można określić odstęp, jaki ma być pozostawiony między poszczególnymi literami podczas pisania. Może to być dowolna wartość całkowita, przy czym 0 oznacza brak odstępu (litery będą się "stykać", jeśli są proporcjonalne).

Funkcja zwraca wartość indeksu, którą należy zapisać w zmiennej, ponieważ będzie ona potrzebna we wszystkich kolejnych kodach odwołujących się do tej strony font, lub zwraca -1, jeśli z jakiegoś powodu funkcja się nie powiedzie.

Warto również zauważyć, że w mapie znaków nie może być żadnych spacji, więc jeśli na przykład używasz znaków "." i "0", ale nie "/", musisz albo dołączyć "/", albo pozostawić pusty obrazek między dwoma używanymi znakami.

Sama strona sprite może być stroną sprite asset z przeglądarki Asset lub załadowaną z pliku przy użyciu strony sprite_add, aby uzyskać indeks załadowanej strony sprite (który jest następnie używany w funkcji). Należy zauważyć, że aby funkcje wyrównywania font (takie jak draw_set_halign()) będą działały zgodnie z oczekiwaniami, gdy używana jest strona sprite font, strona sprite powinna mieć ustawiony początek w lewym górnym rogu. Jeśli zostaną użyte inne wartości, podczas rysowania tekstu za pomocą font trzeba będzie uwzględnić przesunięcie początku.

UWAGA Po załadowaniu strony font do programu GameMaker musisz pamiętać o jej ponownym usunięciu (za pomocą font_delete()), gdy nie jest już potrzebna, w przeciwnym razie istnieje ryzyko wycieku pamięci, który spowolni, a w końcu spowoduje awarię gry.

Składnia:

font_add_sprite(spr, first, prop, sep);

Argument Typ Opis
spr Aktywa Sprite The sprite to add a font based on.
first Real The index of the first available character.
prop Boolean Set as proportional font or not.
sep Real The space to leave between each letter.

Zwroty:

Font Asset

Przykład:

global.Font = font_add_sprite(spr_HUD_Font, ord("!"), true, 2);

Powyższy kod utworzy nową stronę font asset z sprite asset i zapisze indeks nowej strony font w zmiennej "global.Font" do późniejszego wykorzystania.
