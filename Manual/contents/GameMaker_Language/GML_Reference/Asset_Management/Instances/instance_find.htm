instance_find

Wszystkie instancje mają swój unikalny identyfikator (id), który może być używany do ich modyfikowania i manipulowania nimi podczas gry, ale nie zawsze wiesz, jakie jest id konkretnej instancji, więc ta funkcja może być pomocna, ponieważ możesz jej użyć do iteracji po wszystkich instancjach, aby znaleźć to, czego potrzebujesz. Podajesz adres object, którego instancję chcesz znaleźć, oraz liczbę, a jeśli na tej pozycji na liście instancji znajduje się instancja, funkcja zwraca id tej instancji, a jeśli nie, zwraca specjalne słowo kluczowe noone. Możesz także użyć słowa kluczowego all do iteracji po wszystkich instancjach w room, jak również po rodzicu object do iteracji po wszystkich instancjach, które są częścią hierarchii rodzic / dziecko, a nawet możesz określić instancję (jeśli masz jej id) jako sprawdzenie, czy rzeczywiście istnieje w bieżącym room. Zwróć uwagę, że ponieważ instancje są sortowane w arbitralny sposób, nie ma określonej kolejności sprawdzania instancji przez tę funkcję, a każda instancja może znajdować się na dowolnej pozycji.

Maksymalna wartość dla "n" w tej funkcji wyniosłaby

  • Dla słowa kluczowego all: instance_count - 1
  • Dla indeksu object: instance_number(OBJ) - 1

Składnia:

instance_find(obj, n);

Argument Typ Opis
obj Obiekt Aktywa The object to find the nth instance of
n Real The number of the instance to find.

Zwroty:

Instance ID or noone

Przykład:

for (var i = 0; i < instance_number(obj_Enemy); ++i;)
{
    enemy[i] = instance_find(obj_Enemy,i);
}

Powyższy kod użyje strony for loop do iteracji po wszystkich instancjach "obj_Enemy" i przechowa ich id w tablicy "enemy[]".
