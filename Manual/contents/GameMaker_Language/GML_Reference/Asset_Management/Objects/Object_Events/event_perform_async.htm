zdarzenie_perform_async

Funkcja ta służy do wykonywania dowolnego zdarzenia asynchronicznego dostępnego w programie GameMaker. W zmiennej async_load podajesz stałą zdarzenia Async (przedstawioną w poniższej tabeli) oraz mapę DS, która będzie dostępna w wywołanym zdarzeniu Async.

Stała typu zdarzenia asynchronicznego
Stała Opis
ev_async_web_image_load Zdarzenie Załadowano obraz
ev_async_web Zdarzenie HTTP
ev_async_dialog Zdarzenie dialogowe
ev_async_web_iap Wydarzenie związane z zakupem w aplikacji
ev_async_web_cloud Zdarzenie w chmurze
ev_async_web_networking Spotkanie networkingowe
ev_async_web_steam Impreza parowa
ev_async_social Wydarzenie społeczne
ev_async_push_notification Zdarzenie związane z powiadomieniem Push
ev_async_save_load Zapisz/ładuj zdarzenie
ev_async_audio_recording Zdarzenie związane z nagrywaniem dźwięku
ev_async_audio_playback Zdarzenie odtwarzania dźwięku
ev_async_system_event Zdarzenie systemowe

Zdarzenia nieasynchroniczne można wywoływać za pomocą event_perform().

Należy zauważyć, że mapa DS podana w drugim argumencie nie musi być niszczona ręcznie, ponieważ zostanie automatycznie zniszczona przez funkcję po wykonaniu zdarzenia Async.

Składnia:

event_perform_async(type, ds_map);

Argument Typ Opis
type Stała typu zdarzenia asynchronicznego The type of event to perform (see the table above).
ds_map Identyfikator mapy DS The DS map to use as async_load w wywołanym zdarzeniu.

Zwroty:

N/A

Przykład:

var _map = ds_map_create();

_map[? "id"] = "custom_async_event";
_map[? "result"] = true;
_map[? "data"] = { a: 13, b: 16 };

event_perform_async(ev_async_social, _map);

Powyższy kod tworzy mapę DS i wypełnia ją niestandardowymi wpisami, które mają zostać odczytane w zdarzeniu Async. Następnie wykonuje zdarzenie Async Social z nowo utworzoną mapą przekazaną jako async_load dla wywołanego zdarzenia.
