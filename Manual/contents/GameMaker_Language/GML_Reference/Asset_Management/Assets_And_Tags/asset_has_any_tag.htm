asset_has_any_tag

Za pomocą tej funkcji można sprawdzić, czy jeden lub więcej znaczników strings jest przypisanych do dowolnego asset z przeglądarki asset. Podaje się albo nazwę asset (jako string), albo jego indeks asset, a także pojedynczy znacznik string lub tablicę, w której każdy element jest pojedynczym znacznikiem string. Jeśli podasz wartość indeksu asset, musisz podać opcjonalny argument typu asset (stałą), ponieważ assets różnych typów mogą mieć ten sam indeks, choć nie mogą mieć tej samej nazwy. Dostępne typy asset są wymienione w poniższej tabeli:

Stała Opis
asset_object Podana nazwa odnosi się do obiektu.
asset_sprite Podana nazwa odnosi się do sprite'a.
asset_sound Nadana nazwa odnosi się do dźwięku.
asset_room Nazwa odnosi się do room.
asset_tiles Podana nazwa odnosi się do zestawu płytek.
asset_path Podana nazwa odnosi się do ścieżki.
asset_script Nazwa ta odnosi się do script.
asset_font Podana nazwa odnosi się do czcionki.
asset_timeline Imię odnosi się do linii czasu.
asset_shader Podana nazwa odnosi się do shadera.
asset_animationcurve Podana nazwa odnosi się do krzywej animacji.
asset_sequence Podana nazwa odnosi się do sekwencji.

Jeżeli funkcja powiedzie się i co najmniej jeden ze znaczników jest obecny dla adresu asset, zwróci ona adres true, w przeciwnym razie zwróci adres false. Jeśli trzeba sprawdzić dokładne dopasowanie do dowolnego znacznika lub zestawu znaczników, należy użyć funkcji asset_has_tags().

Składnia:

asset_has_any_tag(name_or_index, tags, [asset_type]);

Argument Typ Opis
name_or_index String lub Aktywa The name of the asset (a string) or its index value (an integer).
tags String lub Array ciągów znaków A single asset tag string or an array with various asset tags.
[asset_type] Rodzaj aktywów Stała OPTIONAL! The type of asset to check the tags for, only used when supplying an index for the first argument.

Zwroty:

Boolean

Przykład:

var _a = array_create(3);
_a[0] = "boss";
_a[1] = "main_boss";
_a[2] = "final_boss";

if asset_has_any_tag(object_index, _a, asset_object)
{
    instance_create_layer(0, 0, "Overlay", obj_Boss_Text);
}

Powyższy kod utworzy tablicę znaczników, a następnie sprawdzi, czy któryś z nich został zastosowany do podanej strony object, a jeśli tak, to utworzy instancję innej strony object.
