Struktura toru

Ścieżka struct może być używana zarówno dlaścieżekzasobównajwyższego poziomu , jak i dla ścieżekparametrów podrzędnych , a zachowanie ścieżki określają dwie rzeczy: jej nazwa i typ. Struktury ścieżek są tworzone za pomocą funkcji sequence_track_new() i można je pobrać z witryny sequence assets za pomocą właściwości activeTracks ze struktury Sequence Instance Struct lub właściwości tracks ze struktury Sequence Object Struct.

Właściwości dostępne w ścieżce struct to:

Struktura ścieżki sekwencji
Zmienna Typ Opis
type Typ ścieżki sekwencji Stała Zawiera wartość Stała typu ścieżki sekwencji która opisuje typ ścieżki. Może to być dowolna ze stałych podanych w sekcji "Typ" poniżej.
name String Podczas tworzenia ścieżki "najwyższego poziomu" asset nazwa, którą podajesz, może być dowolna string, która jest potrzebna do identyfikacji ścieżki. Jednak w przypadku ścieżek parametrów musisz podać konkretne strings, aby poinformować GameMakera, jaki rodzaj ścieżki parametrów tworzysz. Więcej informacji na ten temat znajduje się w punkcie "Nazwa" poniżej.
tracks Tablica z Ścieżka sekwencji Strukturas Ta właściwość umożliwia dostęp do listy ścieżek, które są dziećmi tej ścieżki. Przy pobieraniu tej właściwości zwracana jest tablica Struktura ścieżek sekwencjis, a przy ustawianiu tej właściwości należy podać tablicę Struktura ścieżek sekwencjis.
visible Boolean Określa, czy dana ścieżka jest widoczna (wartość true), czy nie (wartość false). Można uzyskać lub ustawić tę wartość, a jeśli ścieżka nie jest widoczna, to żadna z jej ścieżek podrzędnych również nie zostanie narysowana.
keyframes Tablica of sekwencja Keyframe Structs Ta właściwość umożliwia dostęp do listy struktur klatek kluczowych dla danej ścieżki. Przy pobieraniu tej właściwości zwracana jest tablica keyframe structs, a przy ustawianiu należy podać tablicę keyframe structs.

Nazwa

Właściwość name może być dowolnym z poniższych ciągów znaków:

  • "position" - gdy jest ustawione na ścieżce typu seqtracktype_real, oznacza, że zawartość tej ścieżki powinna być używana jako dane pozycyjne.
  • "scale" - ustawione na ścieżce typu seqtracktype_real oznacza, że zawartość tej ścieżki powinna być używana jako dane skalowania.
  • "rotation" - gdy jest ustawione na ścieżce typu seqtracktype_real, oznacza to, że zawartość tej ścieżki powinna być używana jako dane obrotowe.
  • "blend_multiply" lub "image_blend" - ustawiona na ścieżce typu seqtracktype_colour oznacza, że zawartość tej ścieżki powinna być użyta jako dane mnożenia kolorów. Dane keyframe dla takiej ścieżki powinny być tablicą czterech wartości w formacie [A, R, G, B]. Należy pamiętać, że wartości dla każdej składowej są wyrażone w zakresie od 0 do 1, gdzie 0 odpowiada wartości HEX #00, a 1 odpowiada wartości HEX #FF (0 - 255, jak pokazano w oknie wyboru kolorów dla ścieżek mieszania obrazów w edytorze Sequences ).
  • "image_speed" - jeśli jest ustawiony na ścieżce typu seqtracktype_real, oznacza, że zawartość tej ścieżki powinna być używana jako wartość prędkości obrazu.
  • "image_index" - ustawione na ścieżce typu seqtracktype_real oznacza, że zawartość tej ścieżki powinna być używana jako wartość indeksu obrazu.
  • "image_angle" - ustawiony na ścieżce typu seqtracktype_real oznacza, że zawartość tej ścieżki powinna być używana jako wartość kąta obrazu.


    Nazwy parametrów ścieżek tekstowych (używane tylko w przypadku ścieżek tekstowych):
  • "frameSize" - ustawione na ścieżce seqtracktype_real oznacza, że zawartość tej ścieżki powinna być używana jako rozmiar ramki dla tekstu.Każda strona keyframe dla tej ścieżki powinna mieć dwa kanały, 0 i 1, gdzie 0 przechowuje rozmiar X, a 1 - rozmiar Y ramki.
  • "characterSpacing" - używane także w przypadku ścieżek seqtracktype_real, stosowane jako odstępy między znakami tekstu. Każda strona keyframe dla tej ścieżki powinna mieć jeden kanał zawierający wartośćodstępu.
  • "lineSpacing" - używany także w przypadku ścieżek seqtracktype_real, używany jako odstęp między wierszami tekstu. Każda strona keyframe dla tej ścieżki powinna mieć jeden kanał zawierający wartość odstępu.
  • "paragraphSpacing" - używane także ze ścieżkami seqtracktype_real, stosowane jako odstępy między akapitami tekstu. Każda strona keyframe dla tej ścieżki powinna mieć jeden kanał zawierający wartość odstępu.
