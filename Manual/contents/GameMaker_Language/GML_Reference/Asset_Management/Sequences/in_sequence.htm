w_sekwencji

Jest to wbudowana zmienna, która jest częścią zmiennych instancji tworzonych dla każdej instancji object w grze. Jeśli instancja jest kontrolowana przez sequence, ta zmienna zwróci true, w przeciwnym razie zwróci false. Jest to zmienna tylko do odczytu i nie można jej zmieniać.

Należy pamiętać, że ta zmienna stanie się false po zakończeniu kontrolowania Sequence, nawet jeśli jej element Sequence nadal istnieje, i stanie się true, jeśli ten element Sequence zostanie odtworzony ponownie.

Możesz użyć tej zmiennej w kontrolowanej przez gracza (lub CPU) witrynie objects, aby upewnić się, że są one poruszane przez kod tylko wtedy, gdy nie znajdują się w Sequence. Może się to przydać w cutscenkach, gdy musisz użyć tych samych instancji w Sequence (używając funkcji sequence_instance_override_object() ) i trzeba się upewnić, że nie można ich poruszyć zwykłym kodem, gdy jest aktywna dana cutscenka.

Składnia:

in_sequence

Zwroty:

Przykład:

if (!in_sequence)
{
    x += move_x;
    y += move_y;
}

Powyższy kod sprawdza zmienną in_sequence i jeśli jest to false (co oznacza, że instancja nie jest sterowana przez sequence), to dodaje move_x i move_y do pozycji instancji. upewniając się, że porusza się ona tylko wtedy, gdy nie jest sterowana przez Sequence.
