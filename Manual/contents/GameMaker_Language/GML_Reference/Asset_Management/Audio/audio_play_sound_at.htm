audio_play_sound_at

Dzięki tej funkcji możesz odtworzyć dowolny zasób dźwiękowy w danym miejscu w przestrzeni audio. Podajesz indeks dźwięku, a następnie przypisujesz mu pozycję w przestrzeni 3D. Domyślna pozycja słuchacza to (0, 0, 0), więc oznacza to, że jeśli słuchacz nie został przesunięty i chcesz, aby dźwięk dochodził z lewej strony (na przykład), powinieneś ustawić pozycję x na wartość ujemną (więcej informacji na temat ustawiania pozycji słuchacza zob. audio_listener_position()).

Można również ustawić odległość wyłączania (0 powoduje, że dźwięk jest cichy, domyślnie 100), która powoduje, że dźwięk zanika w miarę oddalania się od pozycji słuchacza. To, jak słyszalne jest samo wygaszanie, zależy od wartości odniesienia wygaszania (czyli odległości, poniżej której głośność źródła zmniejszyłaby się o połowę) i współczynnika wygaszania (który wpływa na dźwięk tylko po przekroczeniu odległości odniesienia wygaszania). Domyślnym współczynnikiem jest 1, a efekt różnych wartości falloff zależy od wybranego modelu (pełny przewodnik po różnych modelach falloff i sposobie wykorzystania tych wartości można znaleźć w funkcji audio_falloff_set_model()).

Kolejne dwa argumenty służą do ustawienia, czy dźwięk ma być loop czy nie oraz do przypisania priorytetu dźwiękowi. Priorytet ten jest następnie używany do określenia sposobu postępowania z dźwiękami, gdy liczba odtwarzanych dźwięków przekracza limit ustalony przez funkcję audio_channel_num(). Dźwięki o niższym priorytecie będą zatrzymywane na rzecz dźwięków o wyższym priorytecie, a wartość priorytetu może być dowolną liczbą rzeczywistą (rzeczywista wartość jest dowolna i może wynosić od 0 do 1 lub od 0 do 100, ponieważ GameMaker nada im taki sam priorytet). Zauważ, że kiedy mamy do czynienia z priorytetem, im wyższa liczba tym wyższy priorytet, tak więc dźwięk z priorytetem 100 będzie faworyzowany w stosunku do dźwięku z priorytetem 1.

Funkcja ta zwróci unikalny numer indeksu odtwarzanego dźwięku, który można następnie zapisać w zmiennej, aby można było go wstrzymać lub zatrzymać za pomocą odpowiednich funkcji. Oznacza to, że jeśli masz wiele instancji tego samego dźwięku odtwarzanego w tym samym czasie, możesz skierować tylko jedną instancję tego dźwięku do obsługi za pomocą funkcji audio.

Składnia:

audio_play_sound_at(index, x, y, z, falloff_ref, falloff_max, falloff_factor, loop, priority, [gain], [offset], [pitch], [listener_mask]);

Argument Typ Opis
index Sound Asset The index of the sound to play.
x Real The x position.
y Real The y position.
z Real The z position.
falloff_ref Real The falloff reference relative to the listener (clamp).
falloff_max Real The maximum falloff distance relative to the listener.
falloff_factor Real The falloff factor (default 1).
loop Boolean Flags the sound as looping or not.
priority Real Set the channel priority for the sound.
gain Real OPTIONAL Wartość za zysk.
offset Real OPTIONAL Czas (w sekundach), na który należy ustawić punkt startowy. Wartości dłuższe niż długość podanego dźwięku są ignorowane.
pitch Real OPTIONAL Mnożnik skoku (domyślnie 1).
listener_mask Real OPTIONAL Dane bitmask do ustawienia dla dźwięku. W przypadku celu HTML5 nie będzie to miało żadnego efektu, ponieważ cel nie obsługuje więcej niż jednego słuchacza.

Zwroty:

Sound Instance ID

Przykład 1:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);
}

Powyższy kod sprawdza zmienną globalną "SFX" i jeśli zwróci true to dźwięk zaindeksowany w zmiennej "snd_Waterfall" zostanie zapętlony na swojej room pozycji, z referencją fall-off 100, odległością falloff 300, współczynnikiem falloff 1 i niskim priorytetem.

Przykład 2:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1, 1, 2);
}

Powyższy kod sprawdza zmienną globalną "SFX" i jeśli zwróci true to dźwięk zaindeksowany w zmiennej "snd_Waterfall" zostanie zapętlony na swojej pozycji room z wykorzystaniem podanych ustawień falloff i bez zmiany wzmocnienia. Dźwięk natychmiast zaczyna być odtwarzany od sekundy 2.
