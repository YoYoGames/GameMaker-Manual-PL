audio_play_sound_on

Za pomocą tej funkcji można odtwarzać dowolny zasób dźwiękowy przez emiter, przy czym wszelkie zmiany wzmocnienia emitora, pozycji, wysokości lub prędkości mają wpływ na to, jak użytkownik słyszy końcowy odtwarzany dźwięk. Podajesz indeks emittera do użycia, indeks dźwięku, który ma być odtwarzany, a następnie określasz, czy dźwięk ma być loop czy nie. Następnie można przypisać priorytet, który jest używany do określenia sposobu postępowania z dźwiękami, gdy liczba odtwarzanych dźwięków przekracza limit ustalony przez funkcję audio_channel_num(). Dźwięki o niższym priorytecie będą zatrzymywane na rzecz dźwięków o wyższym priorytecie, a wartość priorytetu może być dowolną liczbą rzeczywistą (rzeczywista wartość jest dowolna i może wynosić od 0 do 1 lub od 0 do 100, ponieważ GameMaker nada im taki sam priorytet). Zauważ, że kiedy mamy do czynienia z priorytetem, im wyższa liczba tym wyższy priorytet, tak więc dźwięk z priorytetem 100 będzie faworyzowany w stosunku do dźwięku z priorytetem 1.

Funkcja ta zwraca również unikalny numer indeksu odtwarzanego dźwięku, który może być przechowywany w zmiennej, aby można było go wstrzymać lub zatrzymać za pomocą odpowiednich funkcji. Oznacza to, że jeśli masz wiele instancji tego samego dźwięku odtwarzanego w tym samym czasie, możesz skierować tylko jedną instancję tego dźwięku do obsługi za pomocą funkcji audio.

Składnia:

audio_play_sound_on(emitter, sound, loop, priority, [gain], [offset], [pitch], [listener_mask]);

Argument Typ Opis
emitter Identyfikator nadajnika audio The index of the emitter to use.
sound Sound Asset The index of the sound to use.
loop Boolean Flags the sound as looping or not.
priority Real Set the channel priority for the sound.
gain Real OPTIONAL Wartość za zysk.
offset Real OPTIONAL Czas (w sekundach), na który należy ustawić punkt startowy. Wartości dłuższe niż długość podanego dźwięku są ignorowane.
pitch Real OPTIONAL Mnożnik skoku (domyślnie 1).
listener_mask Real OPTIONAL Dane bitmask do ustawienia dla dźwięku. W przypadku celu HTML5 nie będzie to miało żadnego efektu, ponieważ cel nie obsługuje więcej niż jednego słuchacza.

Zwroty:

Sound Instance ID

Przykład 1:

if global.SFX
{
    audio_play_sound_on(s_emit, snd_Explode, false, 1);
}

Powyższy kod sprawdza zmienną globalną "SFX" i jeśli zwróci true to dźwięk zaindeksowany w zmiennej "snd_Explode" zostanie odtworzony przez emiter zaindeksowany w zmiennej "s_emit" z niskim priorytetem i bez zapętlenia.

Przykład 2:

if hit == true
{
    audio_play_sound_on(s_emit, snd_Hit, false, 1, 1.3);
}

Powyższy kod sprawdza czy instancja wykonująca kod została trafiona. Jeśli true to odtwarza dźwięk "snd_Hit" przez emitter "s_emit" z nieco większym wzmocnieniem 1.3. Ustawione tutaj wzmocnienie jest mnożone przez wzmocnienie emitera, aby uzyskać ostateczne wzmocnienie, przy którym dźwięk jest odtwarzany.
