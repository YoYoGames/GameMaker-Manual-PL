ścieżka_początkowa

Strona path jest tworzona z serii punktów definiujących, które są ze sobą połączone, a następnie używane do planowania ruchów instancji. Można je utworzyć za pomocą kodu lub w edytorze Path i przypisać do instancji, która ma z nich korzystać w grze. Użyłbyś tej funkcji, by powiedzieć swojej instancji, za którą stroną path ma podążać, z jaką prędkością ma podążać za stroną path (mierzoną w pikselach na krok), jak ma się zachować, gdy dotrze do końca strony path i czy ma podążać za absolutną czy względną pozycją path. Ta ostatnia część oznacza, że instancja albo zaczyna i podąża za path dokładnie tak, jak ją zaprojektowałeś i umieściłeś w edytorze ścieżek (bezwzględnie), albo zaczyna i podąża za path od pozycji, w której instancja została utworzona (względnie).

Jak widać na powyższym obrazku, dwie instancje podążają za tym samym path rozpoczętym od path_start(), ale jedna z nich podąża za absolutnym path, podczas gdy druga za względnym path. Należy zauważyć, że gdy relative jest ustawione na false (absolute), instancja "przeskoczy" na początek path bez względu na to, w którym miejscu room się znajduje.

Stałe używane do definiowania zachowania instancji po osiągnięciu końca strony path są podane na tej stronie.

Składnia:

path_start(path, speed, endaction, absolute);

Argument Typ Opis
path Aktywa na ścieżce The path index to start.
speed Real The speed of which to follow the path in pixels per step, negative meaning going backwards.
endaction Ścieżka Zakończ Działanie Stała What to do when the end of the path is reached (see the constants listed above).
absolute Boolean Whether the calling instance should follow the absolute path as it is in the editor (true) or a relative path to its current position (false).

Zwroty:

N/A

Przykład:

path_start(path, 4, path_action_reverse, 0);

Spowoduje to, że wywołana instancja będzie podążać za stroną path indeksowaną w zmiennej "path" z prędkością 4 pikseli na krok. Będzie podążać za path względem własnej pozycji, a gdy dojdzie do końca, odwróci kierunek i wróci do punktu wyjścia.
