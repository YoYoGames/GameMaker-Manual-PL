network_send_udp

Za pomocą tej funkcji można wysyłać dane przez sieć za pomocą UDP na adres server. Funkcja pobiera id gniazda, przez które ma się łączyć, adres URL, z którym ma się łączyć i port, którego ma używać. Następnie musisz dostarczyć id bufora, który zawiera dane do wysłania (więcej informacji na temat buffers zobacz Reference - Buffers) i wreszcie rozmiar (w bajtach) danych. UDP jest "bezpołączeniowy" w tym sensie, że w rzeczywistości nie wykonujesz połączenia, po prostu wysyłasz pakiet bezpośrednio do IP, a server otrzymuje przychodzące dane z adresu IP i musi sobie z nimi poradzić "jak jest". Funkcja zwróci liczbę bajtów wysłanych danych lub liczbę mniejszą niż 0, jeśli wysyłanie nie powiodło się. Warto zauważyć, że ostateczny rozmiar wysyłanych danych zwracany przez tę funkcję będzie zawierał również informacje z nagłówka GameMaker, czyli dodatkowe 12 bajtów.

Ta funkcja nie będzie działać, gdy jest używana w projekcie uruchomionym na celu HTML5, a także projekty HTML5 nie będą mogły odbierać UDP.

Składnia:

network_send_udp(socket, url, port, bufferid, size);

Argument Typ Opis
socket ID gniazda sieciowego The id of the socket to use.
url String The url or IP to connect to (a string).
port Real The port to connect to.
bufferid ID bufora The id of the buffer to get the data from.
size Real The size (in bytes) of the data.

Zwroty:

Real

Przykład:

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

Powyższy kod wyśle pakiet UDP na adres server zdefiniowany przez URL na port 6510. Dane pobierane są ze strony buffer indeksowanej w zmiennej "buff".
