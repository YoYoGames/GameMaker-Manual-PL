network_send_broadcast

Dzięki tej funkcji można rozgłaszać dane z buffer lokalnie na zakres adresów IP (więcej informacji na temat buffers patrz Odnośnik - Bufory). Zakres jest ograniczony do zakresu urządzenia, na którym działa server, czyli jeżeli urządzenie ma IP 92.168.11.130, to dane zostaną rozesłane na zakres 92.168.11.*. Funkcja zwróci liczbę wysłanych bajtów danych lub liczbę mniejszą od 0, jeżeli wysyłka nie powiodła się.

Ta funkcja będzie działać tylko wtedy, gdy będzie używana z UDP - twój server musi być TCP, a twój klient musi mieć gniazdo klienta UDP utworzone z network_create_socket_ext() aby móc odbierać wszelkie audycje wysyłane z server.

Ta funkcja nie będzie działać, gdy jest używana w projekcie uruchomionym na celu HTML5.

Składnia:

network_send_broadcast(socket, port, bufferid, size);

Argument Typ Opis
socket ID gniazda sieciowego The id of the socket to use.
port Real The port that the server will use.
bufferid ID bufora The id of the buffer to get the data from.
size Real The size (in bytes) of the data.

Zwroty:

Real

Przykład:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

Powyższy kod zapisuje nazwę string bieżącego server (przechowywaną w "global.ServerName"), następnie zapisuje ją do binarnego buffer o id "broadcast_buffer". Dane te są następnie rozgłaszane lokalnie do zakresu IP (IP urządzenia jest obecnie implikowane jako zakres bazowy rozgłaszania) na port 6511.
