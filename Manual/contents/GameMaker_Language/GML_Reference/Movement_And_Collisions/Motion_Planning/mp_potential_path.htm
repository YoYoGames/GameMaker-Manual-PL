mp_potencjalna_ścieżka

Ta funkcja oblicza path dla instancji z jej bieżącej pozycji i orientacji do pozycji określonej przez argumenty xgoal,ygoal. Używa wskazanego rozmiaru kroku, starając się uniknąć kolizji ze wszystkimi instancjami lub tylko tylko tymi, które są oznaczone jako stałe. Podobnie jak w funkcji mp_potential_step() używa do tego celu kroków pola potencjalnego, a sposób, w jaki to robi, można modyfikować za pomocą funkcji mp_potential_settings(). Wskazany adres path musi już istnieć i zostanie nadpisany przez nowy adres path. Funkcja zwraca informację, czy znaleziono path (prawda), czy nie (fałsz).

Aby funkcja nie obliczała w nieskończoność, należy podać współczynnik większy niż 1 - funkcja zatrzyma się i zgłosi awarię, jeśli nie znajdzie adresu path krótszego niż ten współczynnik pomnożony przez odległość między początkiem a celem. Współczynnik 4 jest zazwyczaj wystarczający, ale jeśli spodziewasz się długich objazdów, możesz go wydłużyć. W przypadku niepowodzenia nadal tworzona jest strona path, która biegnie w przybliżonym kierunku do celu, ale do niego nie dotrze.

UWAGA: Ta funkcja nie porusza instancją. Ustawia jedynie adres path, a do poruszania się musisz użyć funkcji Path.

Składnia:

mp_potential_path(path, xgoal, ygoal, stepsize, factor, checkall)

ArgumentTyp Opis
path index of the path that is to be used by the function
xgoal The target x position.
ygoal The target y position.
stepsize The speed the instance moves in pixels per step.
factor This number is used to prevent an infinite loop. It should be larger than 1. See the function description for more information.
checkall Whether to check all instances (true) or just solid ones (false).

Zwroty:

Przykład:

path = path_add();
mp_potential_path(path, obj_Player.x, obj_Player.y, 3, 4, 0);
path_start(path, 3, 0, 0);

Powyższy kod tworzy nowy zasób path i zapisuje jego indeks w zmiennej "path". Następnie generuje stronę path używając mp_potential_path pomiędzy object z tym kodem a object "obj_Player", sprawdzając czy nie ma kolizji room Na koniec uruchamia stronę object wzdłuż tej strony path, nawet jeśli funkcja nie zwróciła kompletnej trasy do "obj_Player".
