file_find_next

Funkcja ta zwraca nazwę następnego pliku spełniającego podaną wcześniej maskę i atrybuty (zdefiniowane przez file_find_first()). Jeśli taki plik nie istnieje, zwracana jest pusta strona string.

OSTRZEŻENIE Ta funkcja może nie działać zgodnie z oczekiwaniami użytkownika, ponieważ GameMaker jest sandboxed! Więcej informacji na ten temat znajduje się w rozdziale System plików.

UWAGA Ta funkcja działa na wszystkich platformach docelowych C++ (Windows, Mac, iOS, Android), ALE filtr flags działa tylko na Windows.

Składnia:

file_find_next();

Zwroty:

Przykład:

if directory_exists("\User Content")
{
    fileA = file_find_first("/User Content/*.doc", fa_readonly);
    fileB = file_find_next();
    fileC = file_find_next();
    file_find_close();
}

Ten kod sprawdza, czy podany katalog istnieje, a jeśli tak, to przechodzi do niego i zwraca pierwszy znaleziony plik "tylko do odczytu" .doc. Następnie szuka dwóch kolejnych plików i zamyka wyszukiwarkę plików.

Można wyszukać dowolną liczbę plików za pomocą while pętli:

var files = [];
var file_name = file_find_first("/User Content/*.doc", fa_readonly);

while (file_name != "")
{
    array_push(files, file_name);

    file_name = file_find_next();
}

file_find_close();

Powyższy kod tworzy pustą tablicę przechowującą wszystkie nazwy plików, które zostały znalezione, i rozpoczyna wyszukiwanie plików tylko do odczytu .doc. Jeśli dana nazwa pliku nie jest pustym string, przesuwa ją do tablicy files, a następnie próbuje znaleźć następny plik. Operacja loop będzie powtarzana do momentu znalezienia pustej tablicy string (co oznacza, że nie ma już pasujących plików), co spowoduje zakończenie operacji loop i zamknięcie wyszukiwarki plików na końcu.
