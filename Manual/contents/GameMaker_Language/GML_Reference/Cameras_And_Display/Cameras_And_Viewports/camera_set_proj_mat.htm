camera_set_proj_mat

Ta funkcja ustawia macierz projekcji dla danej kamery. Użytkownik podaje unikatową wartość ID kamery (zwracaną przez inne camera_create() oraz macierz rzutowania, która ma zostać zastosowana. Więcej informacji na temat tworzenia macierzy projekcji można znaleźć w rozdziale Funkcje macierzowe, a konkretnie na stronach matrix_build_projection_perspective() i matrix_build_projection_ortho().

Podczas korzystania z tej funkcji należy pamiętać o kilku rzeczach:

  • Jeśli Twoja kamera wykonuje automatyczne śledzenie object - tzn. została utworzona przy użyciu camera_create_view() z indeksem object / identyfikatorem instancji, który nie jest równy -1, lub ustawiasz kamerę zdefiniowaną w edytorze Room, lub ustawiasz kamerę domyślną - wtedy ta matryca zostanie nadpisana w następnej klatce gry.
  • Jeśli użyjesz tej funkcji do wyświetlenia strony switch w rzucie perspektywicznym (w odróżnieniu od domyślnego rzutu ortograficznego), automatyczne rysowanie kafelków sprites przestanie działać i będzie wymagało ręcznej obsługi. Dotyczy to wszystkich warstw tła, które wykorzystują kafelkowanie poziome lub pionowe, oraz funkcji draw_sprite_tiled() i draw_sprite_tiled_ext(), które w przypadku użycia rzutu perspektywicznego będą rysować stronę sprite tylko raz, zamiast kafelkować ją.

Składnia:

camera_set_proj_mat(camera_id, matrix)

ArgumentTyp Opis
camera_id The unique camera ID value returned when you created the camera.
matrix The unique matrix ID returned when you created the matrix.

Zwroty:

Przykład:

view_camera[0] = camera_create();
var viewmat = matrix_build_lookat(640, 240, -10, 640, 240, 0, 0, 1, 0);
var projmat = matrix_build_projection_ortho(640, 480, 1.0, 32000.0);
camera_set_view_mat(view_camera[0], viewmat);
camera_set_proj_mat(view_camera[0], projmat);

Powyższy kod tworzy nową kamerę i przypisuje ją do portu widoku[0]. Kamera ta ma następnie ustawione macierze widoku i rzutowania.
