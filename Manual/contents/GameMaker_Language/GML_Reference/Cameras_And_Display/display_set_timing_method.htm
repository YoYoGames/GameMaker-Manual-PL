display_set_timing_method

Za pomocą tej funkcji można ustawić metodę odliczania czasu, która będzie używana do renderowania gry. Metoda może być jedną z poniższych stałych:

Stała Opis
tm_sleep Użyj wartości marginesu uśpienia jako głównej metody pomiaru czasu.
tm_countvsyncs Użyj taktowania vsync jako głównej metody taktowania (domyślnie dla wszystkich obsługiwanych platform)

Metoda taktowania vsync wykorzystuje obsługę synchronizacji pionowej platformy docelowej, aby zapewnić punkt odniesienia dla obliczeń taktowania renderowania gry, podczas gdy ustawienie trybu na margines uśpienia spowoduje, że każda klatka będzie trwała przez odpowiednią ilość czasu (powiedzmy 1/30 lub 1/60 sekundy) poprzez oczekiwanie lub uśpienie. Ogólnie rzecz biorąc, domyślne taktowanie vsync da najbardziej płynne rezultaty, ale należy pamiętać, że podczas korzystania z metody vsync wartości marginesu uśpienia są nadal istotne, choć ich wpływ będzie mniejszy, dlatego zalecamy pozostawienie wartości domyślnej podczas korzystania z tej metody.

Domyślnie na wszystkich platformach z wyjątkiem PS4, Ubuntu i HTML5 program GameMaker będzie używał metody taktowania vsync, natomiast na nieobsługiwanych platformach można używać tylko taktowania sleep margin.

Jeśli chcesz ustawić margines uśpienia, możesz to zrobić za pomocą funkcji display_set_sleep_margin() Za pomocą funkcji można sprawdzić, która metoda odliczania czasu jest aktualnie używana display_get_timing_method().

Składnia:

display_set_timing_method(method);

Argument Typ Opis
method Metoda odmierzania czasu wyświetlania Stała The timing method to use (see the list of constants, above)

Zwroty:

N/A

Przykład:

if display_get_timing_method() != tm_sleep
{
    display_set_timing_method(tm_sleep);
    if display_get_sleep_margin() != 20
    {
        display_set_sleep_margin(20);
    }
}

Powyższy kod sprawdzi metodę timingu, a następnie, jeśli nie jest ustawiona na tm_sleep, zostanie na nią ustawiona, a margines uśpienia zostanie ustawiony na 20.
