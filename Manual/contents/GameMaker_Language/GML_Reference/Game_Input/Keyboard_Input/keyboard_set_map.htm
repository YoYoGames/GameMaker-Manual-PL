keyboard_set_map

Czasami podczas tworzenia gry można sobie zażyczyć, aby jeden klawisz pełnił tę samą funkcję, co inny. Na przykład wiele osób używa klawisza WASD do poruszania się, ale wiele osób używa też klawiszy strzałek! Co więc zrobić? Cóż, można by zakodować system poruszania się dwa razy, ale jest to trochę skomplikowane i na szczęście zbędne, ponieważ ta funkcja pozwala "mapować" jeden klawisz na drugi, dzięki czemu każde polecenie z dowolnego klawisza będzie interpretowane jako takie samo. Aby to zrobić, należy wybrać klawisz, który ma być mapowany (key2 - będzie to klawisz, dla którego napiszesz kod) oraz klawisz, na który ma być on mapowany (klawisz1). Następnie naciśnięcia któregokolwiek z klawiszy będą interpretowane przez GameMaker jako pochodzące z klawisza2. Możesz także użyć tej funkcji do zaprojektowania systemu, w którym użytkownik może zdefiniować własne klawisze do gry, po prostu mapując klawisz wejściowy użytkownika do klawisza, który został zakodowany w grze.

Funkcja przyjmuje wartość keycode zwróconą przez funkcję ord() (tylko wielkie litery od A-Z lub cyfry od 0-9), lub dowolną stałą vk_* wymienioną na głównej stronie Keyboard Input.

WAŻNE! Po użyciu tej funkcji klawisz, na który jest mapowany (klawisz1), nie będzie już mógł być używany jako jego rzeczywisty symbol. Na przykład, jeśli strzałka w górę zostanie zmapowana na "W", naciśnięcie klawisza "W" nie będzie już rozpoznawane jako "W". klawisza "W" jako "W", będzie on zawsze traktowany jako strzałka w górę. Aby to cofnąć, należy albo zmapować klawisz na sam siebie - tak więc zarówno klawisz1, jak i klawisz2 będą miały adres ord("W") - albo użyć funkcji keyboard_unset_map().

Składnia:

keyboard_set_map(key1, key2);

ArgumentTyp Opis
key1 This is the key that key1 is to be mapped to
key2 This is the key that is to be mapped

Zwroty:

Przykład:

keyboard_set_map(ord("A"), vk_left);

Powyższy przykładowy kod mapuje klawisz "A" na klawisz strzałki w lewo. Oznacza to, że gracz może używać albo klawisza "A", albo klawisza strzałki w lewo, a cały kod napisany dla strzałki w lewo będzie również reagował na użycie klawisza "A" zamiast niej.
