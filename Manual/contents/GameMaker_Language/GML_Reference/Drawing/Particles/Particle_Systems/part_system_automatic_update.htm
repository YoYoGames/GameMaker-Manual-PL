część_systemu_automatycznej_aktualizacji

Funkcja ta kontroluje, czy program GameMaker powinien automatycznie aktualizować system cząsteczek, czy nie. W normalnych warunkach nie ma potrzeby korzystania z tej funkcji, ponieważ domyślna wartość true (automatyczna aktualizacja jest włączona) jest tym, czego oczekujesz. Jednak w przypadku efektów specjalnych lub wstrzymywania gry możesz ustawić tę wartość na false, co uniemożliwi przeprowadzanie jakichkolwiek aktualizacji w danym systemie cząsteczek, chyba że zostanie to wyraźnie nakazane za pomocą funkcji part_system_update() lub ponownie użyjesz tej funkcji, aby włączyć automatyczne aktualizacje. Brak aktualizacji oznacza, że cząstki utworzone w tym systemie nie będą już zmieniać położenia, koloru ani żadnego innego parametru, a emitery również przestaną działać, "zamrażając" system dokładnie w tym momencie, w którym automatyczna aktualizacja została ustawiona na false.

Należy pamiętać, że w przypadku korzystania z prostych efektów utworzonych za pomocą funkcji effect_create_above() lub effect_create_below() to jako indeksu systemu cząsteczek można użyć wartości 0 (dla efektów poniżej) lub 1 (dla efektów powyżej) i w ten sposób również dla nich włączyć automatyczną aktualizację (działa to również w przypadku włączania aktualizacji wizualnych efektów cząsteczkowych GML ).

Składnia:

part_system_automatic_update(ind, automatic);

Argument Typ Opis
ind Identyfikator systemu cząstek stałych The index of the particle system to change.
automatic Boolean Whether automatic updating is on (true) lub nie (fałsz).

Zwroty:

N/A

Przykład:

if global.Pause
{
    part_system_automatic_update(global.Sname, false);
}

Powyższy kod switch wyłączy aktualizacje cząsteczek, jeśli zmienna globalna "Wstrzymane" będzie miała wartość true.
